# 実装方針（#400）

配信終了処理はすでにここです。

```
StreamSessions::EndService
```

そして終了ボタンの controller は

```
Cast::StreamSessionsController#finish
```

ここで現在はこうなっています。

```ruby
redirect_to cast_booth_path(ended_session.booth_id), notice: "スタンバイを終了しました"
```

これを **result画面へリダイレクト** に変更します。

---

# 変更コード

## `app/controllers/cast/stream_sessions_controller.rb`

### 変更前

```ruby
def finish
  ended_session = StreamSessions::EndService.new(
    stream_session: @stream_session,
    actor: current_user
  ).call

  redirect_to cast_booth_path(ended_session.booth_id), notice: "スタンバイを終了しました"
rescue => e
  redirect_to cast_booth_path(@stream_session.booth_id), alert: e.message
end
```

---

### 変更後

```ruby
def finish
  ended_session = StreamSessions::EndService.new(
    stream_session: @stream_session,
    actor: current_user
  ).call

  redirect_to cast_stream_session_path(ended_session),
              notice: "今回の配信が終了しました"
rescue => e
  redirect_to cast_booth_path(@stream_session.booth_id), alert: e.message
end
```

このあと **小さなUX改善が1つあります（2分で終わる）**。

今の実装だと

```
終了 → redirect
```

なので **配信映像が一瞬残る可能性**があります。

---
原因はそのままで、**#400 で仕様を変えたのに、既存integration testの期待値が旧仕様のまま** だからです。

失敗しているのはこのテストです。

```ruby
test "standby: finish ends session and redirects to summary" do
```

ここで今は

```ruby
assert_redirected_to cast_booth_path(@booth)
```

を期待していますが、#400 で `finish` の遷移先を

```ruby
cast_stream_session_path(session)
```

に変更したので、**テストだけが古い** 状態です。

---

## 修正方針

このテスト名と期待値を、新仕様に合わせて直せばOKです。

### 修正前

```ruby
test "standby: finish ends session and redirects to summary" do
  session = StreamSessions::StartService.new(booth: @booth, actor: @cast).call
  assert @booth.reload.standby?

  post finish_cast_stream_session_path(session)
  assert_response :redirect
  assert_redirected_to cast_booth_path(@booth)

  @booth.reload
  assert @booth.offline?
  assert_nil @booth.current_stream_session_id
end
```

### 修正後

```ruby
test "standby: finish ends session and redirects to result" do
  session = StreamSessions::StartService.new(booth: @booth, actor: @cast).call
  assert @booth.reload.standby?

  post finish_cast_stream_session_path(session)
  assert_response :redirect
  assert_redirected_to cast_stream_session_path(session)

  @booth.reload
  assert @booth.offline?
  assert_nil @booth.current_stream_session_id
end
```

---

## なぜ `session` のままでよいか

`finish` 後でも `StreamSession` レコード自体は残るので、URL はその same id で問題ありません。
`ended_session` が返ってきても、実質同じ `stream_session.id` です。

---

## テスト観点としてはむしろ良くなっている

この修正後テストは、

* 終了処理が通る
* booth は offline になる
* current_stream_session_id が外れる
* 遷移先が result 画面になる

をまとめて確認できるので、#400 の仕様に合っています。
