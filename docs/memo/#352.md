# 実装プラン

今回やることは 3 つです。

1. `_ivs_viewer.html.erb` に Overlay 骨格を追加する
   映像と同じ partial 内に `#viewer_overlay` を置き、viewer 用 slot を並べます。
   `#viewer_flash_layer` は既存どおり `#viewer_stage` 直下に残し、Overlay とは分離します。

2. `_viewer.scss` に viewer 用 Overlay 基盤を追加する
   live 側の class 設計に寄せつつ、viewer 専用名で独立させます。
   `grid` と `clamp()` で row / slot サイズを調整できるようにし、debug 属性で development 時だけ枠表示できるようにします。

3. 今回は slot の中身は枠だけに留める
   meta / drink / comment / ops / drink action は、後続 Issue の実装対象です。
   今回は位置確認のためのラベルだけを置きます。
   既存の offscreen UI はそのまま残します。

# 補足

今回の実装で満たしていることは次のとおりです。

* Overlay が Stage 上に absolute 配置される
* meta / drink / comment / ops / drink action の slot が揃う
* Overlay 全体が Stage 内に収まる
* ページスクロールは増やさない
* `clamp()` で各 row / slot の調整ができる
* development 時だけ debug 背景とラベルが出る
* mute ボタンは既存 `ivs_viewer_controller.js` をそのまま利用する

一方で、今回まだやっていないものは意図どおりです。

* meta の本表示
* コメントパネル本体
* ドリンク表示本体
* drink action row の本体
* ops slot の panel 化
* 未配信時 UI
