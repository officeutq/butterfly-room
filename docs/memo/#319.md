## 実装プラン（高レベル）

1. **HTML（live.html.erb）**

* `#live_stage` の中に `#live_overlay` を追加（`#live_media` と同階層）
* 現在 `.live-offscreen` に入っている “映像以外のDOM” を、`#live_overlay` 内へ移動し、**見えるコンテナ**として配置

  * Turbo Stream の target / id は **そのまま保持**（`flash_inner`, `cast-pending-drink-orders-container`, `cast_comment_section`, `comments`, `comment_form`, `cast_stream_meta` など）
* `footer#app_footer` は今のまま（Stage 高さ計算は live_stage_controller が `app_footer` を参照していて整合している）

2. **CSS（application.scss）**

* `#live_overlay` を `position:absolute; inset:0;` で Stage 全面
* Overlay 自体は `pointer-events:none`
* Overlay 内の “実UIコンテナ” は `pointer-events:auto`
* 重なり順：`#live_media` より上にする（z-index）
* まずは “暫定で全部出す” なので、Overlay 内コンテナは **画面全体を使える**ようにし、必要なら軽い余白だけ付ける

### 2) `app/assets/stylesheets/application.scss`

**追加**：Overlay レイヤーの基盤CSSを `#live_stage` 近辺に追記します（live にだけ効く）。

下記を `#live_stage` / `#live_media` のスタイル群の近くに追加してください。

```scss
/* Overlay 레イヤー（映像上にDOMを重ねる） */
#live_overlay {
  position: absolute;
  inset: 0;

  /* 映像より前面 */
  z-index: 10;

  /* 映像操作を邪魔しない：操作したい要素だけ auto にする */
  pointer-events: none;
}

/* 実UIコンテナだけ操作可能にする */
#live_overlay .live-overlay-ui {
  pointer-events: auto;

  /* 暫定：全体を使えるようにしつつ、最低限の余白 */
  padding: 12px;

  /* 画面いっぱいに広げる（配置は次Issueで詰める） */
  width: 100%;
  height: 100%;

  /* たくさん出すと溢れるので暫定で内部スクロール可（次Issueで“ページスクロール無し”へ寄せる） */
  overflow: auto;
}
```

※ここだけは “暫定で全部出す” に合わせて `overflow: auto` を入れています。
Epic #315 的には「ページスクロール無し」へ寄せたいので、次の #320/#321/#322 で各パネル固定＋内部スクロールにしたタイミングで、`overflow` を外す/制御するのが自然です。
