以下の方針で進めれば、Issue #486 のスコープに収まります。

## 実装プラン

1. `effects` テーブルを追加する
   カラムは Issue 本文どおり `name`, `key`, `zip_filename`, `icon_path`, `enabled`, `position`。
   `key` と `zip_filename` には unique index を張ります。
   `enabled` は `true`、`position` は `0` を default にします。

2. `Effect` モデルを追加する
   `name`, `key`, `zip_filename` の presence、`key`, `zip_filename` の uniqueness、`position` の数値性を持たせます。
   scope は `enabled_only`, `ordered` を追加します。

3. 今回は controller / view / JS / routes は触らない
   `none` と `beauty` は別枠のまま維持し、DB は追加 Banuba Effect のメタ情報だけを持つ前提です。

---

## 追加・変更ファイル

### 1. migration 作成コマンド

まず空 migration を作成してください。

```bash
docker compose exec app bin/rails generate migration CreateEffects
```

---

### 2. `db/migrate/XXXXXXXXXXXXXX_create_effects.rb`

生成された空ファイルを、以下全文で置き換えてください。

```ruby
class CreateEffects < ActiveRecord::Migration[8.1]
  def change
    create_table :effects do |t|
      t.string :name, null: false
      t.string :key, null: false
      t.string :zip_filename, null: false
      t.string :icon_path
      t.boolean :enabled, null: false, default: true
      t.integer :position, null: false, default: 0

      t.timestamps
    end

    add_index :effects, :key, unique: true
    add_index :effects, :zip_filename, unique: true
    add_index :effects, %i[enabled position]
  end
end
```

---

### 3. `app/models/effect.rb`

新規作成してください。

```ruby
# frozen_string_literal: true

class Effect < ApplicationRecord
  validates :name, presence: true
  validates :key, presence: true, uniqueness: true
  validates :zip_filename, presence: true, uniqueness: true
  validates :position, numericality: { only_integer: true }

  scope :enabled_only, -> { where(enabled: true) }
  scope :ordered, -> { order(:position, :id) }
end
```
