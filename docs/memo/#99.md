## #99 の前提整理（いまの状態）

いま揃っているもの：

* EC2 上で Rails（Puma）が **[http://0.0.0.0:3000](http://0.0.0.0:3000)** で起動中
* SG で 3000/TCP が（少なくとも検証用に）開いている
* RDS / S3 / ActiveStorage OK

👉 **CloudFront から EC2:3000 に HTTP で流すだけ**で成立する。

---

## 方針

### #99 でやること

* CloudFront **Distribution を1つ作る**
* Origin = **EC2 の Public IPv4 / Public DNS**
* Viewer protocol = **HTTPS only**
* 証明書 = **CloudFront デフォルト証明書**
* キャッシュ設定 = 最小（ほぼ無効）

### #99 でやらないこと

* ALB（#100）
* 独自ドメイン
* WAF
* キャッシュ最適化

---

## 手順（AWS コンソール）

### EC2 → Elastic IP → Elastic IP アドレスを割り振る（任意）

* 👉 割り振る
* Elastic IP を選んで、**アクション → Elastic IP アドレスを関連付ける**
* リソースタイプ: インスタンス
* インスタンス: `butterfly-room-app-1`
* プライベート IP: そのまま（1個しかないはず）
* 👉 関連付ける

### ALB を作成
* #100 の ⑤ まで実施する

### CloudFront → ディストリビューション → 作成


#### 1️⃣ Choose a plan
* `$0`

#### 2️⃣ Get started
* Distribution name：`butterfly-room-ec2`
* Description - optional：空欄でOK
* Distribution type：Single website or app（そのまま）
* Domain（Route 53 managed domain - optional）：空欄のまま

#### 3️⃣ Specify origin
* Origin type：`Other`
* Custom origin：`butterfly-room-alb-620286126.ap-northeast-1.elb.amazonaws.com`
* Origin path - optional：空
* オリジン設定
  * プロトコル：HTTP のみ（まずはこれでOK）
* Cache settings
  * ビューワープロトコルポリシー：`Redirect HTTP to HTTPS`
  * 許可された HTTP メソッド：`GET, HEAD, OPTIONS, PUT, POST, PATCH, DELETE`
  * キャッシュポリシー：`CachingDisabled`
  * オリジンリクエストポリシー - オプション：`AllViewer`

#### 4️⃣ Enable security
* なしでOK（あとで付けられる）

#### 5️⃣ Review and create

→ Create distribution
