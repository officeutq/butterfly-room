了解です。
Issue #377 は、**既存DOMを壊さずに visible 領域から退避し、viewer main 内に Stage 用空間を確保する**ことに集中して実装するのがよいです。

今回の実装方針は次です。

* `show.html.erb` の **縦積み stub / お気に入り / 退室** も visible 領域から退避する
* `#stream_state` は **固定 wrapper として維持**
* `_stream_state.html.erb` では

  * **visible 領域**として `viewer_stage` を残す
  * presence / meta / drink menu / pending drink orders / comments / comment form は **退避領域**へ移す
* 退避は `display: none` ではなく、**DOM を維持したまま画面外へ逃がすクラス**で行う
* `_ivs_viewer.html.erb` は今回は大きく触らず、既存の viewer 本体として使う

以下、そのままコピペできる完成形です。

## 1. `app/views/booths/show.html.erb`

```erb
<% content_for :head do %>
  <meta name="turbo-cache-control" content="no-cache">
<% end %>

<div class="viewer-page">
  <div class="viewer-legacy-offscreen" aria-hidden="true">
    <h1>Public Booth#show (stub)</h1>

    <p>booth_id: <%= @booth.id %></p>

    <%# ★お気に入りボタン（stream_state とは分離して固定DOM） %>
    <% if user_signed_in? %>
      <%= render "favorites/booths/button_booth_show", booth: @booth, favorited: @booth_favorited %>
    <% end %>
  </div>

  <%# ★booth単位で常時購読（開始/終了/状態変化を自動反映） %>
  <%= turbo_stream_from [@booth, :stream_state] %>

  <%# ★箱（wrapper）は show 側に固定。idを絶対に消さない %>
  <div id="stream_state">
    <%= render "booths/stream_state",
          booth: @booth,
          stream_session: @stream_session,
          comments: @comments,
          drink_items: @drink_items %>
  </div>

  <div class="viewer-legacy-offscreen" aria-hidden="true">
    <hr>

    <div class="d-flex justify-content mb-3">
      <%= link_to "退室", root_path, class: "btn btn-outline-secondary" %>
    </div>
  </div>
</div>
```

## 2. `app/views/booths/_stream_state.html.erb`

```erb
<%# locals: booth:, stream_session:, comments:, drink_items: %>

<% live_like = booth.live? || booth.away? %>

<div class="viewer-stream-state">
  <% if stream_session.present? && live_like %>
    <%= turbo_stream_from [stream_session, :pending_drink_orders] %>
    <%= turbo_stream_from [stream_session, :comments] %>
    <%= turbo_stream_from [stream_session, :viewer_stage] %>

    <div class="viewer-stage-shell">
      <div id="viewer_stage">
        <%= render "stream_sessions/ivs_viewer", stream_session: stream_session %>
      </div>
    </div>

    <div class="viewer-legacy-offscreen" aria-hidden="true">
      <div
        data-controller="presence-poll"
        data-presence-poll-ping-url-value="<%= ping_stream_session_presence_path(stream_session) %>"
        data-presence-poll-summary-url-value="<%= presence_summary_stream_session_path(stream_session) %>"
        data-presence-poll-interval-ms-value="20000"
      >
        <% title_display = stream_session.title.presence || booth.name %>

        <%# 配信タイトル・説明 %>
        <div class="mb-2">
          <div class="fw-semibold"><%= title_display %></div>
          <% if stream_session.description.present? %>
            <div class="text-muted small"><%= stream_session.description %></div>
          <% end %>
        </div>

        <hr>

        viewers: <span data-presence-poll-target="count">-</span>
      </div>

      <hr>

      <%= render "booths/drink_menu",
            booth: booth,
            stream_session: stream_session,
            drink_items: drink_items %>

      <hr>

      <%= render "stream_sessions/pending_drink_orders", stream_session: stream_session %>

      <hr>

      <div id="comments">
        <%= render partial: "comments/comment", collection: (comments || []), as: :comment %>
      </div>

      <%= render "stream_sessions/comments/form", stream_session: stream_session %>
    </div>

  <% else %>
    <div class="viewer-stage-shell viewer-stage-shell-empty">
      <div id="viewer_stage"></div>
    </div>

    <div class="viewer-legacy-offscreen" aria-hidden="true">
      <div class="text-muted">現在配信していません</div>

      <hr>

      <%= render "booths/drink_menu",
            booth: booth,
            stream_session: stream_session,
            drink_items: drink_items %>

      <hr>

      <div class="text-muted">配信中のみ表示されます</div>

      <hr>

      <p>配信中のみコメントできます</p>
    </div>
  <% end %>
</div>
```

## 4. 今回の実装で満たせること

* viewer の既存縦積み要素が visible 領域から退避される
* `#stream_state` / `#viewer_stage` / `#comments` / `drink_menu` / `pending_drink_orders` などの DOM を維持できる
* `turbo_stream_from` と Stimulus 接続を壊しにくい
* viewer main 内に Stage 用の空き領域を確保できる
* live / away / offline の分岐を保ったまま次 Issue に渡せる
