# #97 S3バケット作成（production用）最短手順

## ゴール

* Rails production が **ActiveStorage + S3** で正常起動する
* EC2 には **IAM Role** でS3アクセス（キー直書きなし）
* `.env.production` に必要最小限だけ書く
* #98 と同時進行推奨

---

## ① S3バケットを作成

AWSコンソール → **S3**

### バケット作成

* **バケット名**

  ```
  butterfly-room-production
  ```

  （※ 一意ならOK。名前変えたら後半も揃える）

* **リージョン**

  ```
  アジアパシフィック（東京） ap-northeast-1
  ```

* **パブリックアクセス**

  * ❌ すべてブロック（デフォルトのまま）

* **その他**

  * 変更不要 → 作成

✅ これでOK（ポリシーやACLは後回しでよい）

---

## ② EC2用 IAM Role を作る（超重要）

### 1) IAM → ロール → ロールを作成

* **信頼されたエンティティ**

  * AWSサービス
  * **EC2**

### 2) 許可ポリシーを付与

一旦これでOK（後で絞れる）👇

* `AmazonS3FullAccess`
  （最低限にするなら後述するけど、今はスピード優先）

### 3) ロール名

```
butterfly-room-ec2-role
```

---

## ③ EC2 に IAM Role をアタッチ

EC2 → 対象インスタンス →
**アクション → セキュリティ → IAMロールを変更**

* `butterfly-room-ec2-role` を選択 → 保存

👉 **再起動不要**（即反映）

---

## ④ Rails 側の設定確認

### `config/storage.yml`

これは **そのままで正解** 👌

```yml
amazon:
  service: S3
  access_key_id: <%= ENV.fetch("AWS_ACCESS_KEY_ID", nil) %>
  secret_access_key: <%= ENV.fetch("AWS_SECRET_ACCESS_KEY", nil) %>
  region: <%= ENV.fetch("AWS_REGION", "ap-northeast-1") %>
  bucket: <%= ENV.fetch("AWS_S3_BUCKET") %>
```

※ IAM Role 使うので
`AWS_ACCESS_KEY_ID / SECRET` は **未設定でOK**

---

## ⑤ `.env.production` を整理（重要）

### ✅ 正しい最小構成（例）

```env
RAILS_ENV=production
NODE_ENV=production
RAILS_LOG_TO_STDOUT=1

DATABASE_URL=postgres://butterfly_room:****@butterfly-room-postgres.cviquuwo4fu6.ap-northeast-1.rds.amazonaws.com:5432/butterfly_room_production

RAILS_MASTER_KEY=8xxxxxxxxxxxxxxxx
SECRET_KEY_BASE=xxxxxxxxxxxxxxxxx

AWS_REGION=ap-northeast-1
AWS_S3_BUCKET=butterfly-room-production
```

#### SECRET_KEY_BASE が未設定なら

EC2で一度だけ：

```bash
docker compose -f docker-compose.production.yml run --rm app \
  bin/rails secret
```

出てきた値を `.env.production` に貼る。

---

## ⑥ 再ビルド & 起動

```bash
docker compose -f docker-compose.production.yml --env-file .env.production down
docker compose -f docker-compose.production.yml --env-file .env.production build --no-cache
docker compose -f docker-compose.production.yml --env-file .env.production up -d
```

ログ確認👇

```bash
docker logs -f butterfly_room-app
```

### 期待するログ

* ❌ `Missing service adapter for "S3"` → 出ない
* ❌ `missing required option :name` → 出ない
* ❌ `InstanceProfileCredentials::Non200Response` → 出ない
* ✅ `Rails application starting in production` で **落ちずに待機**
