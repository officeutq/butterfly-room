了解です。
結論からいうと、#402 は **実装 Issue ではなく検証 Issue** ですが、**今回の検証前提を満たすための最小修正は1点だけ必要**です。

## 結論

#402 でやることは次の2つです。

1. **検証前提を揃える最小修正**
2. **既存 setter を Console から叩いて安全範囲を決める**

## なぜ最小修正が必要か

今のコードでは `setLipsSize()` が `0..1` clamp になっています。
しかし、今回確定した UI 前提はこうです。

* 口スライダー値 `x` は `-1..1`
* `lipsSize = x`
* `lipsMouthSize = -x`

このままだと、左側の検証で `lipsSize = -1` を再現できません。
なので、**#402 の検証を正しく行うために `setLipsSize()` の clamp だけ直す**のが妥当です。

目については、

* `eyeRounding = x`
* `eyeEnlargement = x`
* `x = 0..1`

なので、既存 setter のままで検証できます。

# 観測ポイント

各値で次を確認します。

## 見た目

* 不自然な変形がないか
* 顔追従が破綻しないか

* プレビューが明らかにカクつかないか
* 配信開始後に著しく重くならないか
* 長時間で悪化しないか

## 安定性

* 値変更後にフリーズしないか
* 配信継続中に Stage が不安定にならないか
* 終了後にカメラやマイクが残らないか

# 安全範囲の決め方

#402 では「Banuba が理論上動く最大」ではなく、**配信 UX として安全な範囲**を決めます。

採用基準はこうです。

* 見た目が不自然にならない
* 顔追従が安定
* 配信中に明らかなカクつきが出ない
* 長時間でも崩れにくい

つまり、1.0 まで動いても、見た目や安定性が悪ければ **UI 最大値はそこまで開けない** 方針です。
