## (4) Issue #282 に貼る「候補リスト（追加/不要判断）」案

そのまま Issue の「追加対象候補（TODO）」に貼れる形です。

* `/admin/cast_metrics` (`admin_cast_metrics_path`)：**追加**

  * 理由：store_admin がキャスト売上を日常的に確認する画面。`require_current_store!` のため store 未選択時は店舗選択へ誘導。
* `/admin/drink_items` (`admin_drink_items_path`)：**追加**

  * 理由：ドリンクメニューは店舗運用で編集頻度が高い。`require_current_store!` のため store 未選択時は店舗選択へ誘導。
* `/admin/store_bans` (`admin_store_bans_path`)：**追加**

  * 理由：店舗運用でBAN管理が必要。`require_current_store!` のため store 未選択時は店舗選択へ誘導。
* `/admin/settlements/:id` (`admin_settlement_path`)：**不要**

  * 理由：`/admin/settlements` の一覧に「詳細」リンクがあり、一覧→詳細で到達できる（ダッシュボードに置かない）。
* `/admin/booths/:id` / `/admin/booths/:id/edit` / `/admin/booths/:id/watch`：**今回はダッシュボード追加はしない**

  * 理由：設計としてはブース管理（一覧/詳細）から到達する想定。現状 `admin/booths/index` に詳細導線が薄いため、別Issueで「一覧→詳細（管理操作）」導線を整備する。
