実装方針は最小差分です。

* `_ivs_viewer.html.erb` から `aspect-ratio: 16 / 9` を撤廃
* viewer 用のラッパーに **表示領域の高さ** と **黒背景** を持たせる
* `video` 自体はその領域いっぱいに広げつつ、`object-fit: contain` で等比表示
* ミュートUIの DOM / target / action はそのまま維持
* `ivs_viewer_controller.js` は今回は変更不要

これで #336 の Done 条件である

を満たす構成です。

## `ivs_viewer_controller.js` について

今回は変更不要です。
理由は、今回触るのが

* `data-ivs-viewer-target="video"`
* `muteButton`
* `muteHint`

の **見た目の配置だけではなく DOM の存在自体はそのまま** だからです。
`toggleMute()` や `srcObject` の付け替えにも影響しません。

---

## この実装の意図

以前は `video` 自体に `aspect-ratio: 16 / 9` が入っていたので、受信映像が縦長でも、表示枠が先に 16:9 へ固定されていました。
今回はその固定をやめ、代わりに

* 親 `.ivs-viewer-stage` が表示領域を持つ
* 子 `video` はその中で `contain`

にすることで、**受信映像の比率を壊さず黒帯込みで収める** 形にしています。
