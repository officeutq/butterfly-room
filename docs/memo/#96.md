## 1. RDS 作成画面へ

**RDS → データベース → データベースの作成**


### データベース作成方法を選択

* ✅ **フル設定**

（簡単作成は NG。ネットワーク制御できない）

### エンジンのオプション

エンジンのタイプ

* ✅ **PostgreSQL**

バージョン

* PostgreSQL 15.15-R1

👉 迷ったら **15 系**でOK。

RDS 延長サポートの有効化：なし

### テンプレート

* ✅ **開発/テスト**

（本番でも最初はこれでOK。後でスケール）

### 可用性と耐久性

* ✅ **マルチ AZ DB インスタンスデプロイ (2 インスタンス)**

### 設定（超重要）

DB インスタンス識別子

```
butterfly-room-postgres
```

マスターユーザー名

```
butterfly_room
```

認証情報管理

* **AWS Secrets Manager で管理 - 最も安全にチェック**

暗号化キーを選択
* 👉 **AWS/SecretsManager（デフォルト）**（推奨）

> 後で Rails の `DATABASE_URL` に使う

その他の認証情報の設定
* ✅ **パスワード認証**

### インスタンス設定

DB インスタンスクラス

* 開始時おすすめ：

  * `db.t4g.micro`（Arm）
  * or `db.t3.micro`

👉 最初は **micro で十分**。

### ストレージ

| 項目       | 値    |
| -------- | ---- |
| ストレージタイプ | gp3  |
| 割り当て     | 20GB |
| 自動スケーリング | オフ   |

### 接続（ここが一番大事）

コンピューティングリソース
* ✅ EC2 コンピューティングリソースに接続しない

ネットワークタイプ
* ✅ IPv4

Virtual Private Cloud (VPC)
* ✅ `butterfly-room-vpc`

DB サブネットグループ

* **`butterfly-room-db-subnet-group`**

パブリックアクセス

* ❌ **なし**

VPC セキュリティグループ

* 既存の選択

既存の VPC セキュリティグループ

* ❌ default は外す
* ✅ **`butterfly-room-rds-sg` のみ**

RDS Proxy
* チェックしない

認証機関

* ✅ デフォルト（rds-ca-rsa2048-g1）

### モニタリング

* ✅ データベースインサイト - スタンダード

Performance Insights
* ✅ 有効

保持期間
* ✅ 7日

AWS KMS キー
* ✅ (default) aws/rds

### その他のモニタリング設定
拡張モニタリング
* ✅ 有効

OS メトリクスの詳細度
* 60秒

OS メトリクスのモニタリングロール
* デフォルト

ログのエクスポート
* ⛔ iam-db-auth-error ログ → OFF
* ✅ PostgreSQL ログ → ON
* ⛔ アップグレードログ → OFF

DevOps Guru
* オフ
