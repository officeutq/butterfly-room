## (1) High-level implementation understanding（タスク抽出＋順序）

### タスク（順序）

1. **ルーティング追加**

   * `GET /admin/settlements`（index）
   * `GET /admin/settlements/:id`（show）
     既存の `admin` 名前空間に追加。

2. **controller追加（Admin::SettlementsController）**

   * `before_action :require_current_store!`
   * `index`:

     * 当月概算（current_store の当月 `StoreLedgerEntry.occurred_at` 合計 × 0.7 floor + carryover）
     * 1万円未満の場合は「繰越予定」表示を優先（参考値は小さく）
     * 履歴：`settlements` を `status in (confirmed, exported, paid)` に固定して一覧
   * `show`:

     * store向けに **期間・status・金額・確定/生成日時**のみ
     * 口座情報・CSV・操作履歴は **出さない**

3. **view追加（index/show）＋文言固定（誤認防止）**

   * NGワードを絶対に含めない（残高/引出/出金/ウォレット/預り金/いつでも…）
   * 注記の固定文言を入れる（概算、月次、翌月10日、1万円未満繰越、運営側処理で変動）

4. **ダッシュボード導線（store_admin に精算リンク追加）**

   * `store_admin` セクションに「精算（予定・履歴）」リンクを追加
   * `current_store` 未選択時は `admin_stores_path` に誘導（既存パターン踏襲）

5. **テスト追加（integration）**

   * store_admin: 自店舗のみ（他店舗の settlement が表示されない）
   * draft が表示されない
   * 禁止ワードが含まれない（簡易テキスト検証）
   * current_store 未選択なら `admin_stores_path` へリダイレクト
   * system_admin も閲覧できる（ただし表示データは current_store のみ＝店舗表示としての混同回避）

### 2) app/controllers/admin/settlements_controller.rb（新規作成）

```ruby
# frozen_string_literal: true

module Admin
  class SettlementsController < Admin::BaseController
    before_action :require_current_store!
    before_action :set_settlement, only: %i[show]

    ZONE = "Asia/Tokyo"
    MIN_PAYOUT_YEN = 10_000
    SHARE_RATE = BigDecimal("0.7")

    def index
      # --- 1) estimate (this month, SSOT: StoreLedgerEntry.occurred_at) ---
      Time.use_zone(ZONE) do
        @month_from = Time.zone.today.beginning_of_month.beginning_of_day
        @month_to = Time.zone.now

        gross_yen =
          StoreLedgerEntry
            .where(store_id: current_store.id, occurred_at: @month_from...@month_to)
            .sum(:points)
            .to_i

        @month_gross_yen = gross_yen
        @month_share_yen = calc_share(gross_yen)

        @carryover_yen =
          SettlementCarryover
            .where(store_id: current_store.id)
            .sum(:amount_yen)
            .to_i

        @estimated_payable_yen = @month_share_yen + @carryover_yen
        @below_min_payout = @estimated_payable_yen < MIN_PAYOUT_YEN
      end

      # --- 2) history (store-facing: confirmed/exported/paid only) ---
      @settlements =
        current_store
          .settlements
          .where(status: %i[confirmed exported paid])
          .order(period_from: :desc, id: :desc)
    end

    def show
      # store-facing: do NOT expose payout snapshot, exports, or events
      unless %w[confirmed exported paid].include?(@settlement.status)
        head :not_found
      end
    end

    private

    def set_settlement
      @settlement = current_store.settlements.find(params[:id])
    end

    def calc_share(gross_yen)
      (BigDecimal(gross_yen) * SHARE_RATE).floor(0).to_i
    end
  end
end
```

ポイント：

* `show` は draft を見せないために `not_found` 扱いにしています（store側に内部状態を出さない）。
* system_admin も admin 配下に入れる設計なので、閲覧できても **current_store で絞られる**＝店舗表示として混同しにくいです。
