## 実装方針

順番はこの通りが安全です。

1. 空マイグレーション生成
2. migration全文を手書き
3. controller の strong params から `:description` 削除
4. model の `validates :description` 削除
5. migrate 実行
6. `description` 残存確認
7. 動作確認

## 1. 空マイグレーション生成

```bash
docker compose exec app bin/rails generate migration RemoveDescriptionFromStreamSessions
```

生成されたファイルを開いて、全文を以下に置き換えてください。

```ruby
class RemoveDescriptionFromStreamSessions < ActiveRecord::Migration[8.0]
  def up
    remove_column :stream_sessions, :description, :text
  end

  def down
    add_column :stream_sessions, :description, :text
  end
end
```

## 2. `app/controllers/cast/stream_sessions_controller.rb`

`metadata_params` を以下に変更してください。

```ruby
    def metadata_params

    end
```

## 3. `app/models/stream_session.rb`

全文は以下です。

```ruby
class StreamSession < ApplicationRecord
  belongs_to :booth
  belongs_to :store
  belongs_to :started_by_cast_user, class_name: "User"
  has_many :presences, dependent: :destroy

  enum :status, { live: 0, ended: 1 }

  validates :title, length: { maximum: 64 }, allow_nil: true

  delegate :current_stream_session_id, :status, to: :booth, prefix: true
end
```

## 5. 残存確認コマンド

まず app 配下の残存確認です。

```bash
docker compose exec app bash -lc 'grep -R -F "stream_session.description" -n app'
```

```bash
docker compose exec app bash -lc 'grep -R -F "stream_session[description]" -n app'
```

```bash
docker compose exec app bash -lc 'grep -R -F ":description" -n app/controllers app/models'
```

最後に schema 確認です。

```bash
docker compose exec app bash -lc 'grep -n "description" db/schema.rb'
```

ここで `stream_sessions` の `description` が消えていればOKです。
