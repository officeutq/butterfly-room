## #62 最短手順（Windows → Docker → Rails）

### 1) AWS CLI v2 を Windows にインストール

AWS公式のインストール手順：

まず公式ページを開いて **AWS CLI v2 (MSI installer)** を入れます。
（インストール後、PowerShell を開き直す）
https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html

### 2) PowerShell を開き直して確認

```powershell
aws --version
```

これが `aws-cli/2.x.x ...` と出ればOK。

---

### 3) AWS 認証情報を作る（ローカル開発用）

1. AWSコンソール > IAM > ユーザー > ユーザーの作成
   * ユーザー名：butterfly-room-local
   * AWS マネジメントコンソールへのユーザーアクセスを提供する：チェックしない
2. 許可を設定
   * 許可のオプション：ポリシーを直接アタッチする
   * ポリシーの作成 → JSON → 以下を入力
      ```
      {
        "Version": "2012-10-17",
        "Statement": [
          {
            "Sid": "IVSRealTimeStage",
            "Effect": "Allow",
            "Action": [
              "ivs:CreateStage",
              "ivs:DeleteStage",
              "ivs:GetStage",
              "ivs:ListStages"
            ],
            "Resource": "*"
          },
          {
            "Sid": "IVSRealTimeParticipant",
            "Effect": "Allow",
            "Action": [
              "ivs:CreateParticipantToken"
            ],
            "Resource": "*"
          }
        ]
      }
      ```
      → 次へ
      * ポリシーの詳細
          * ポリシー名：`ButterflyRoomIvsRealTimeDev`
          → ポリシーの作成 → 許可を設定に戻る
   * 許可ポリシー：`ButterflyRoomIvsRealTimeDev` を選択して、次へ → ユーザーの作成
3. IAM > ユーザー > butterfly-room-local
    * アクセスキーを作成 → ローカルコード → 次へ → ButterflyRoomIvsRealTimeDev → アクセスキーを作成

```powershell
aws configure
# AWS Access Key ID 
# AWS Secret Access Key 
# Default region name: ap-northeast-1
# Default output format : json
```

---

### 4) 認証が通ったか確認（これが超重要）

```powershell
aws sts get-caller-identity
```

Account / Arn が返れば成功です。

## 5) Windows の認証情報を Docker（Rails）に渡す（おすすめ：~/.aws をマウント）

AWS CLI の設定が入る場所は通常 `C:\Users\<あなた>\.aws` です。
これをコンテナに **read-only マウント**します。

`docker-compose.yml` の `app` サービスに追加（例）：

```yaml
services:
  app:
    environment:
      AWS_REGION: ap-northeast-1
      AWS_PROFILE: default
    volumes:
      - ${USERPROFILE}\.aws:/root/.aws:ro
```

> これで Rails（aws-sdk）も同じ認証情報を使えるので、`169.254.169.254`（EC2メタデータ）を見に行かなくなります。

## 6) 疎通確認（Rails から本物の Stage 作成）

あなたの現状だと、配信開始（StartService）で Stage 作成を呼ぶので、まずここで確認できます。

```bash
docker compose exec app bash -lc "bin/rails c"
```

```rb
booth = Booth.find(1)
actor = User.find(2) # cast
session = StreamSessions::StartService.new(booth: booth, actor: actor).call
session.reload.ivs_stage_arn
```

ここで `arn:aws:ivsrealtime:ap-northeast-1:...:stage/...` みたいなのが入れば #62 の「Stage作成疎通」はOK。
