はい。#589 で**ソースを書く前に先に揃えるべきこと**は、まずこの順番です。

**1. DeepAR のアカウント作成**
DeepAR Web SDK は、使う前に **developer portal でアカウントを作る**前提です。公式 docs でも、まず developer portal でアカウントを作成し、そこからプロジェクトと Web App を作る流れになっています。([docs.deepar.ai][1])

**2. プロジェクト作成**
portal に入ったら、新しい project を作ります。Web の getting started では、`+ New Project` から project を作成し、無料プランを選ぶ流れが案内されています。テストや小規模用途向けの無料ライセンスがあることも明記されています。([docs.deepar.ai][2])

**3. Web App を追加して、localhost 用のドメインを登録**
次に project の中で **Web App** を追加します。このとき DeepAR は **URL ではなく domain name を登録する**方式です。公式 docs では、プロトコルやパスを含めずに domain を入れるよう案内しています。また、license key は **production だけでなく localhost でも必要**です。([docs.deepar.ai][2])

この Issue だと、まずは **localhost 用の登録**を先にやるのがよいです。
実運用ドメインは後で追加すればよく、#589 では Rails の `localhost:3000` で動けば十分です。

**4. License Key を取得して保管する**
Web App を追加すると、その app 用の **license key** を使うことになります。DeepAR はこの key で利用可否や usage control を行う仕組みです。#589 では、まずこの key を `.env` に入れる前提で手元に確保しておくところまでが事前準備です。([docs.deepar.ai][3])

この段階では、環境変数名を先に決めておくと後が楽です。
たとえば方針としては **`DEEPAR_LICENSE_KEY`** で揃えるのが自然です。

**5. 導入方式を決める**
DeepAR Web は大きく **NPM導入** と **script/CDN導入** の2系統があります。公式 getting started にもその2方式が案内されています。#589 は「まず Rails 上で検証画面を作る」Issue なので、最初は **CDN 読み込みで PoC** に寄せるのが一番軽いです。([docs.deepar.ai][2])

この判断を先にしておくと、

* importmap に pin するのか
* 外部 script を layout に置くのか
* SDK 本体を `public/...` に置くのか
  がブレません。

#589 の時点では、**CDN で始める**のが妥当です。

**6. effect の入手先を決める**
DeepAR は初期化しただけだと plain camera feed で、見た目を変えるには **effect ファイル** を読み込む必要があります。公式 docs では、effect は通常 `.deepar` ファイルで、Web ではアプリから fetch できる場所に置く必要がある、とされています。effect の切替は `switchEffect` API で行います。([docs.deepar.ai][4])

ここで決めるべきことは2つです。

* **最初は公式サンプル effect を使うか**
* **自前で `public/deepar/effects/...` に置くか**

#589 では、まずは **公式サンプル effect で動作確認** が最短です。
自前配置は次の段階でも大丈夫です。

**7. SDK追加資材を CDN 任せにするか、自前配信にするか決める**
Banuba と違って、DeepAR は WebAssembly や ML model などの追加ファイルを内部で使いますが、**デフォルトでは JsDelivr CDN から自動取得**します。さらに `rootPath` を指定すれば、自前サーバー配信にも切り替えられます。公式 docs では、速度最適化や圧縮のためには最終的に自前配信が推奨されています。([docs.deepar.ai][5])

なので判断はこうです。

* **#589 検証段階**：CDN 任せでよい
* **本組み込みや最適化段階**：`public/...` 配置や自前 CDN を検討

つまり、今すぐ Banuba と同じように
`public/deepar/sdk/...` を全部用意する必要はありません。
ただし、**将来的には自前配信に寄せる余地がある**、という理解が正しいです。([docs.deepar.ai][5])

**8. カメラ起動方針を決める**
DeepAR Web はデフォルトで **前面カメラを使い、必要時にカメラ権限を要求**します。一方で、`disableDefaultCamera` を使えば **自前の video source** を流し込む構成にもできます。`startCamera()` を後から呼ぶ構成も公式にあります。([docs.deepar.ai][1])

#589 では IVS とまだつながないので、先に決めるべき方針はこれです。

* **今回は DeepAR 単独でカメラを起動する**
* IVS 用の media 管理とは分離する
* 後続 Issue で custom video source 化を検討する

これで PoC と本組み込みの責務が混ざりません。

**9. 動作確認のチェック項目を先に決める**
実装前に、成功条件を先にメモしておくと迷いません。今回の Issue の Done に合わせるなら、最低限こうです。

* license key で初期化できる
* localhost:3000 で起動する
* カメラ権限が取れる
* effect が見た目に反映される
* 数分動かして console error が出ない
* Turbo 遷移前後でカメラが残らない

これらは DeepAR の初期化、effect 読み込み、camera handling の仕様にそのまま対応しています。([docs.deepar.ai][6])

## 4. 「構造を崩していないか」を確認する

ここが大事です。
DeepAR は **追加資材を `rootPath` 基準で解決**するので、`node_modules/deepar` の中身を public に置くときは、**構造を崩さない**のが安全です。([DeepAR Documentation][1])

つまり考え方はこうです。

* `node_modules/deepar/js/...` → `public/deepar/js/...`
* `node_modules/deepar/wasm/...` → `public/deepar/wasm/...`
* `node_modules/deepar/models/...` → `public/deepar/models/...`

のように、**ルートだけ `public/deepar` に変わる**状態にします。

## 4. `app/javascript/controllers/deepar_verification_controller.js`

新規追加です。
`pin_all_from "app/javascript/controllers"` が入っているので、**importmap の追加設定は不要**です。

```javascript
import { Controller } from "@hotwired/stimulus"

export default class extends Controller {
  static targets = [
    "preview",
    "error",
    "state",
    "faceState",
    "startBtn",
    "stopBtn",
    "effectLabel",
  ]

  static values = {
    licenseKey: String,
    rootPath: String,
    defaultEffectUrl: String,
  }

  connect() {
    this._deepAR = null
    this._started = false
    this._selectedEffectUrl = this.defaultEffectUrlValue || ""

    this._beforeCache = () => this.stop()
    document.addEventListener("turbo:before-cache", this._beforeCache)

    this._setState("idle")
    this._setFaceState("face: unknown")
    this._syncEffectLabel()
    this._syncUI()
  }

  disconnect() {
    document.removeEventListener("turbo:before-cache", this._beforeCache)
    this.stop()
  }

  async start() {
    if (this._started) return

    if (!this.licenseKeyValue) {
      this._setError("DEEPAR_LICENSE_KEY が未設定です。")
      return
    }

    if (!window.deepar || typeof window.deepar.initialize !== "function") {
      this._setError("DeepAR SDK の読み込みに失敗しました。/deepar/js/deepar.js を確認してください。")
      this._setState("error")
      return
    }

    this._clearError()
    this._setState("starting")

    try {
      const deepAR = await window.deepar.initialize({
        licenseKey: this.licenseKeyValue,
        previewElement: this.previewTarget,
        effect: this._selectedEffectUrl,
        rootPath: this.rootPathValue,
      })

      if (deepAR?.callbacks) {
        deepAR.callbacks.onFaceVisibilityChanged = (visible) => {
          this._setFaceState(visible ? "face: visible" : "face: hidden")
        }
      }

      this._deepAR = deepAR
      this._started = true
      this._setState("running")
      this._clearError()
    } catch (error) {
      console.error("[deepar-verification] start failed", error)
      this._setError(this._humanizeError(error))
      this._setState("error")
      this._shutdown()
    } finally {
      this._syncUI()
    }
  }

  stop() {
    if (!this._deepAR && !this._started) {
      this._setState("idle")
      this._setFaceState("face: unknown")
      this._syncUI()
      return
    }

    this._setState("stopping")

    try {
      this._shutdown()
      this._clearError()
      this._setState("idle")
      this._setFaceState("face: unknown")
    } catch (error) {
      console.error("[deepar-verification] stop failed", error)
      this._setError("停止処理でエラーが発生しました。再読込してください。")
      this._setState("error")
    } finally {
      this._syncUI()
    }
  }

  async selectEffect(event) {
    const nextUrl = event.currentTarget?.dataset?.effectUrl
    if (!nextUrl) return

    this._selectedEffectUrl = nextUrl
    this._syncEffectLabel()
    this._syncEffectButtons(event.currentTarget)

    if (!this._deepAR) return

    this._clearError()
    this._setState("switching-effect")

    try {
      await this._deepAR.switchEffect(nextUrl)
      this._setState("running")
    } catch (error) {
      console.error("[deepar-verification] switchEffect failed", error)
      this._setError("effect の切替に失敗しました。")
      this._setState("error")
    } finally {
      this._syncUI()
    }
  }

  _shutdown() {
    if (this._deepAR && typeof this._deepAR.shutdown === "function") {
      this._deepAR.shutdown()
    }

    this._deepAR = null
    this._started = false

    if (this.hasPreviewTarget) {
      this.previewTarget.innerHTML = ""
    }
  }

  _syncUI() {
    if (this.hasStartBtnTarget) {
      this.startBtnTarget.classList.toggle("d-none", this._started)
      this.startBtnTarget.disabled = this._started
    }

    if (this.hasStopBtnTarget) {
      this.stopBtnTarget.classList.toggle("d-none", !this._started)
      this.stopBtnTarget.disabled = !this._started
    }
  }

  _syncEffectLabel() {
    if (this.hasEffectLabelTarget) {
      this.effectLabelTarget.textContent = this._selectedEffectUrl || "(none)"
    }
  }

  _syncEffectButtons(activeButton) {
    this.element.querySelectorAll("[data-effect-url]").forEach((button) => {
      const isActive = button === activeButton
      button.classList.toggle("btn-primary", isActive)
      button.classList.toggle("btn-outline-primary", !isActive)
    })
  }

  _setState(state) {
    if (this.hasStateTarget) {
      this.stateTarget.textContent = state
    }
  }

  _setFaceState(state) {
    if (this.hasFaceStateTarget) {
      this.faceStateTarget.textContent = state
    }
  }

  _clearError() {
    if (this.hasErrorTarget) {
      this.errorTarget.textContent = ""
    }
  }

  _setError(message) {
    if (this.hasErrorTarget) {
      this.errorTarget.textContent = message
    }
  }

  _humanizeError(error) {
    const name = error?.name
    const message = `${error?.message || error}`

    if (name === "NotAllowedError" || name === "SecurityError") {
      return "カメラ権限が拒否されました。ブラウザ設定で許可してください。"
    }

    if (name === "NotFoundError" || name === "OverconstrainedError") {
      return "利用できるカメラが見つかりません。接続やOS設定を確認してください。"
    }

    if (name === "NotReadableError") {
      return "カメラを使用できません。他アプリが使用中の可能性があります。"
    }

    return `DeepAR の起動に失敗しました: ${message}`
  }
}
```

## 5. 手順

### 反映

`docker compose` で app を起動し直してください。

```bash
docker compose restart app
```

`restart` で足りなければこれです。

```bash
docker compose down
```

```text
http://app.localhost:3000/dev/deepar_verification
```

* ページが開く
* 「起動」でカメラ許可ダイアログが出る
* プレビューが表示される
* `Aviators` / `Lion` / `Koala` で見た目が変わる
* 顔が見えると `face: visible` に変わる
* 「停止」でカメラが止まる

## 補足

今回の実装は、DeepAR docs の `initialize({ licenseKey, previewElement, effect })` と self-hosted 用 `rootPath` の考え方に合わせています。自前ホスト時は `rootPath` を SDK ルートに向け、JS と SDK ルートのバージョン一致・構造維持が重要です。`shutdown()` で再初期化可能な状態に戻せます。([DeepAR Documentation][1])

ここまで入れて、エラーが出たらそのエラー全文を貼ってください。
