## 1. `app/views/cast/stream_sessions/_pending_drink_orders.html.erb`

```erb
<% pending = DrinkOrder
  .includes(:drink_item)
  .where(stream_session_id: stream_session.id, status: :pending)
  .order(:created_at, :id) %>

<div id="cast_pending_drink_orders">
  <% if pending.empty? %>
    <div class="text-muted small">未消化のドリンクはありません</div>
  <% else %>
    <% head_id = pending.first.id %>

    <div class="vstack gap-2">
      <% pending.each do |order| %>
        <% if order.id == head_id %>
          <%= button_to consume_cast_drink_order_path(order),
                method: :post,
                class: "cast-pending-drink-order-btn" do %>
            <%= drink_item_display(order.drink_item, klass: "cast-pending-drink-order-display") %>
          <% end %>
        <% else %>
          <div class="cast-pending-drink-order-disabled">
            <%= drink_item_display(order.drink_item, klass: "pending-drink-item-display") %>
          </div>
        <% end %>
      <% end %>
    </div>
  <% end %>
</div>
```

## 4. 見た目の意図

今回の追加クラスの役割はこうです。

* `cast-pending-drink-order-btn`

  * 先頭要素全体を押下可能にする
  * hover / active で「押せそう」を出す

* `cast-pending-drink-order-display`

  * 先頭だけ少し輪郭を立てる

* `cast-pending-drink-order-disabled`

  * 非先頭は押下不可
  * cursor は default

---

このあと確認するなら、見るポイントは次の3つです。

1. 先頭だけ hover / active が出るか
2. consume 後に次の item が先頭表示へ切り替わるか
3. Turbo Stream 更新後もボタン全体押下が維持されるか
