## (1) Implementation plan

1. **customer の booths#show を booth単位で常時購読に切り替える**

   * `turbo_stream_from [@booth, :stream_state]` を常時追加
   * `div#stream_state` を追加して、ここだけを差し替え対象に固定

2. **`booths/_stream_state` を新設して表示を集約**

   * **live/away のときだけ** viewer / comments / pending_drink_orders / presence を表示
   * standby/offline（または session なし）は **未配信と同じ見え方**にする（＝スタンバイ表示はしない）

3. **booth状態が変わる瞬間に stream_state を broadcast する**

   * “開始”＝`standby -> live` の瞬間なので、`Cast::BoothsController#status` の成功後に broadcast
   * “終了”＝EndService の commit 後に broadcast（booth が offline & current_session=nil になるため）

4. **既存の ended broadcast は維持（必要なら後で整理）**

   * いまの `broadcast_ended` は stream_session 単位の UI 差し替え（黒画面防止等）として残しつつ、
   * customer 画面の「終了後UI」は booth単位の stream_state 差し替えで確実に反映させる

### A-3) `app/notifiers/stream_session_notifier.rb`（追記）

```ruby
# frozen_string_literal: true

class StreamSessionNotifier
  def self.broadcast_stream_state(booth:)
    booth = Booth.find(booth.id)
    Turbo::StreamsChannel.broadcast_replace_to(
      [booth, :stream_state],
      target: "stream_state",
      partial: "booths/stream_state",
      locals: {
        booth: booth,
        stream_session: booth.current_stream_session,
        comments: booth.current_stream_session ? Comment.alive.where(stream_session: booth.current_stream_session)
                                               .order(created_at: :desc).limit(50).reverse : []
      }
    )
  end

  def self.broadcast_ended(stream_session)
    Turbo::StreamsChannel.broadcast_replace_to(
      [ stream_session, :pending_drink_orders ],
      target: "pending_drink_orders",
      partial: "stream_sessions/ended",
      locals: { stream_session: stream_session }
    )

    Turbo::StreamsChannel.broadcast_replace_to(
      [ stream_session, :cast_pending_drink_orders ],
      target: "cast_pending_drink_orders",
      partial: "cast/stream_sessions/ended",
      locals: { stream_session: stream_session }
    )

    Turbo::StreamsChannel.broadcast_replace_to(
      [ stream_session, :viewer_stage ],
      target: "viewer_stage",
      partial: "stream_sessions/viewer_ended",
      locals: { stream_session: stream_session }
    )
  end
end
```

> ※コメントの取り方は `BoothsController#show` と同じに寄せています（最小で動く形）。

### A-5) `app/services/stream_sessions/end_service.rb`（booth を持ち出して broadcast）

差分イメージ（transactionの外で booth を使えるようにする）：

```ruby
def call
  authorize!

  ended_session = nil
  ended_booth = nil

  StreamSession.transaction do
    session = StreamSession.lock.find(@stream_session.id)
    raise AlreadyEnded if session.ended_at.present?

    booth = Booth.lock.find(session.booth_id)

    booth.update!(
      status: :offline,
      current_stream_session_id: nil
    )

    DrinkOrders::RefundService.new(stream_session: session).call!

    session.update!(ended_at: Time.current, status: :ended)

    ended_session = session
    ended_booth = booth
  end

  StreamSessionNotifier.broadcast_ended(ended_session)

  # ★Issue42: booth単位のUIを「未配信」に自動切替（リロード不要）
  StreamSessionNotifier.broadcast_stream_state(booth: ended_booth)

  ended_session
end
```

## (5) Pitfalls / verification steps

### Pitfalls

* customer の `presence_summary` は `authenticate_user!` なので、未ログインだと viewers 数は更新されません（これは仕様としてOKならOK）。
* `booths/_stream_state` で `turbo_stream_from` を live/away のときだけにしているので、**standby 中は購読も走らず完全に未配信**になります（意図通り）。
* ended 時に `broadcast_ended` だけだと booth.show は切替わらないので、**EndService 側の booth broadcast は必須**です（今回追加済み）。

### Verification steps（Doneに直結）

1. customer（ログイン済み推奨）で `booths#show` を開く（offline 画面のまま）。
2. cast で「スタンバイ開始」→ cast live画面へ（customer は **未配信のまま**であること）。
3. cast で「配信開始」→ publisher が `_patchBoothStatus("live")` を叩く
   → customer の `stream_state` が **自動で配信UIに切替**すること（リロード不要）。
4. cast で「配信終了」
   → customer の `stream_state` が **自動で未配信表示に戻る**こと（リロード不要）。

必要なら、このまま **Issue42用のPR文（原因/対応/確認手順/Done）**もコピペ用に作れます。
