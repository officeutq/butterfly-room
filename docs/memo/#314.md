## 最終実装プラン（過不足なく・最小変更）

### 目的

* 共通ヘッダーを `sticky-top` で常時表示
* 将来の可視領域計測のため `id="app_header"` / `id="app_footer"` を付与
* Epic #286 の「機能非改変」を守り、UIの属性追加のみで完結（JSロジック追加なし）

### 変更点（ファイル別）

1. `app/views/layouts/application.html.erb`

* `<header>` に

  * `id="app_header"` を追加
  * `sticky-top` を追加
* 共通 `<footer>` に

  * `id="app_footer"` を追加

2. `app/views/cast/booths/live.html.erb`

* live側 `<footer>` に `id="app_footer"` を追加

  * ※ layout 側 footer は `live_layout_moved` で描画されないため id 重複なし

3. CSS（原則変更なし）

* まずは `sticky-top` のみで確認
* もし「本文がヘッダーに潜る」不具合が出た場合のみ、**追加Issueに分けるか**、このIssue内で “最小のpadding調整” を入れる（ただし現状の仕様には明記がないので、まずは調整なしで進めるのが安全）
