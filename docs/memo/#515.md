補足すると、これでやっていることは3つだけです。

* `sessionStorage` に viewer の mute 状態を保存する
* `connect()` 時にその状態を復元する
* `_addStreams()` で `video.muted` の現在値ではなく、**保存済みのユーザー選好** を使う

これで `live -> away -> live` のたびに controller が再生成されても、視聴側のスピーカー状態は引き継がれます。

一点だけ大事なのは、初回のデフォルトです。
この差分では **初回は muted=true で開始** になります。
もし「初回だけは unmute を優先したい」を維持したいなら、デフォルト値だけこう変えます。

```diff
-    this._mutedPreference = true
+    this._mutedPreference = false
```

ただしその場合、ブラウザの autoplay 制約で失敗したら catch 側で `true` が保存されるので、その後は muted 優先になります。
今回の #515 を最優先で確実に潰すなら、まずは **初回 muted=true** の方が安全です。

必要なら次に、**初回だけ unmute 優先を残しつつ、live/away では保持する版** に整えます。
