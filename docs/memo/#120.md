了解。**ブランチは不要**で、#120 は **EC2 上で “queue DB に SolidQueue テーブルを作る”**だけです。手順はこれだけ。

---

## #120 手順（EC2 / production）

### 0) まず、今のコンテナが動いている前提

```bash
cd ~/apps/butterfly-room
docker compose -f docker-compose.production.yml ps
```

---

### 1) （念のため）primary 側の migrate

すでにやってるならスキップ可。

```bash
docker compose -f docker-compose.production.yml exec app bin/rails db:migrate
```

---

### 2) runner で queue_schema.rb を読み込む（破壊判定を踏まない）

`db:schema:load:queue` を使わずに、**queue 接続に切り替えて schema をロード**します。

```bash
docker compose -f docker-compose.production.yml exec app \
  bin/rails runner "ActiveRecord::Base.establish_connection(:queue); load Rails.root.join('db','queue_schema.rb')"
```

これで `solid_queue_*` テーブルが作られます（必要な分だけ）。

---

### 3) テーブルができたか確認（solid_queue_jobs）

```bash
docker compose -f docker-compose.production.yml exec app \
  bin/rails runner "ActiveRecord::Base.establish_connection(:queue); puts ActiveRecord::Base.connection.table_exists?('solid_queue_jobs')"
```

`true` が出ればOK。
