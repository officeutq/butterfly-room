## 実装方針（最小差分 / #227）

### 変更点の要約

* `Admin::StoresController#index`

  * **system_admin**：`Store` 全件を表示
  * **store_admin**：従来どおり「admin membership を持つ store のみ」
* `Admin::CurrentStoresController#create`

  * **system_admin**：membership チェックなしで任意の store を選択可能（ただし **Store が存在すること**は確認）
  * **store_admin**：従来どおり membership 必須
* テスト追加（integration）

  * system_admin：一覧が全件表示される
  * store_admin：自分の store のみ
  * current_store 未選択でも `/admin/stores` に到達でき、選択導線が成立する（system_admin / store_admin 両方）

> `Admin::BaseController#current_store` の system_admin fallback（`Store.first`）は #206 の対象なので **今回は触りません**。

## 期待される挙動（Doneチェックに対応）

* ✅ system_admin で `/admin/stores` に **全 store 表示**
* ✅ store_admin の表示範囲は **従来どおり**
* ✅ current_store 未選択でも `/admin/stores` に入れて、`この店舗を開く` で選択が完了し、`admin_root_path` へ遷移できる
* ✅ `Admin::DashboardController` の `require_current_store!` と競合しない（store_admin は選択後に到達、system_admin はそもそもガード対象外）

---
原因、これです。

`Admin::BaseController#current_store` が **system_admin でも `session[:current_store_id]` を“無効扱いして消している”** せいで、`/admin/stores` 再表示時に `@current_store_id` が空になり、`table-active` が付かなくなっています。

### 何が起きてるか（現状ロジック）

`current_store` 内のここ：

```ruby
if session[:current_store_id].present?
  store = Store.find_by(id: session[:current_store_id])
  if store.present? && admin_membership_exists_for_store?(store.id)
    return store
  end

  session.delete(:current_store_id)
end
```

* system_admin は StoreMembership を持たない運用なので `admin_membership_exists_for_store?` が常に false
* その結果、**session[:current_store_id] が毎回消される**
* layout 等で `current_store` が呼ばれた時点で消えるので、テストの「table-active」が成立しない

---

## 修正（最小差分）

system_admin の場合だけ、`session[:current_store_id]` は **「存在するstoreなら有効」**にします（membershipチェックしない）。

### `app/controllers/admin/base_controller.rb` の該当部分だけ置き換え

`# 1) session 優先` のブロックをこの形に変更してください：

```ruby
      # 1) session 優先
      if session[:current_store_id].present?
        store = Store.find_by(id: session[:current_store_id])

        if store.present?
          # system_admin は membership を持たない運用なので、存在する store ならOK
          return store if current_user.system_admin?

          # store_admin は admin membership がある store のみ有効
          return store if admin_membership_exists_for_store?(store.id)
        end

        # 改ざん/脱退/削除 への耐性：不正ならクリア
        session.delete(:current_store_id)
      end
```

これで

* system_admin の選択が **session に残る**
* store_admin は従来どおり membership 必須
* “存在しない store_id” は system_admin でも消える（改ざん耐性は維持）
