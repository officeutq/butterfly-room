実装方針です。

* `comments/_comment.html.erb` を、Issue #468 の基準DOMに沿って「アイコン列 + テキスト列」へ変更します。
* 名前と本文は右カラム内でインラインフローにし、折り返しは右カラム左端から始まるようにします。
* アイコンと名前は既存 helper を使い、表示ロジックは増やしません。
* 共通 partial を変更するため、`_viewer.scss` だけでなく `_live_comment.scss` にも同じ comment 内部レイアウトを足して、cast/live 側でも崩れないようにします。
* 既存の投稿フォーム、Turbo Stream target、コメント一覧コンテナ構造は変更しません。

`app/views/comments/_comment.html.erb`

```erb
<div class="comment comment-item">
  <div class="comment-avatar">
    <%= user_avatar_badge(comment.user, size: 32, klass: "comment-avatar-badge") %>
  </div>

  <div class="comment-text">
    <span class="comment-author"><%= display_name_or_anonymous(comment.user) %></span>
    <span class="comment-body"><%= comment.body %></span>
  </div>
</div>
```

この場合、SCSS は両方とも `viewer-comment-*` / `live-comment-*` ではなく、配下で `.comment-item`, `.comment-avatar`, `.comment-text`, `.comment-author`, `.comment-body`, `.comment-avatar-badge` を定義してください。
共通 partial を使っている現状だと、こちらの中立クラス版のほうが実運用ではおすすめです。

その中立クラス版で進めるなら、差し替え後の `_viewer.scss` と `_live_comment.scss` もまとめて出します。
