# [Ops] VPC / サブネット / SG 作成手順（最小構成）

> 目的：
> EC2（Rails）＋ RDS（PostgreSQL）＋ CloudFront Origin を置く
> **「あとで壊さずに拡張できる最小ネットワーク」**を作る

---

## 全体構成（最終形のイメージ）

```
Internet
  ↓
CloudFront
  ↓ (HTTPS)
EC2 (Public Subnet)
  ↓ (SG制限)
RDS (Private Subnet)
```

* CloudFront は **VPC外**
* EC2 は **Public Subnet**
* RDS は **Private Subnet**
* 外部公開は CloudFront 経由のみ（EC2直アクセス最小）

---

## ① VPC 作成

### AWS コンソール

**VPC → VPC → VPCを作成**

| 項目        | 値                  |
| --------- | ------------------ |
| 作成するリソース | VPC のみ |
| 名前タグ - オプション | butterfly-room-vpc |
| IPv4 CIDR ブロック | IPv4 CIDR の手動入力 |
| IPv4 CIDR | `10.0.0.0/16`      |
| IPv6 CIDR ブロック | IPv6 CIDR ブロックなし |
| テナンシー     | デフォルト              |
| VPC 暗号化コントロール | なし |

👉 VPC を作成

> `/16` にしておくと、後で ALB / Redis / Bastion を足しても余裕があります。

## ⑤ Security Group 作成

### ⑤-1 EC2 用 SG

**EC2 → セキュリティグループ → セキュリティグループを作成**

| 項目  | 値                     |
| --- | --------------------- |
| セキュリティグループ名  | butterfly-room-ec2-sg |
| 説明 | Security group for butterfly-room EC2 (Rails app). Allows HTTP from CloudFront and SSH from operator. |
| VPC | butterfly-room-vpc    |

#### インバウンドルール

| タイプ  | ポート | ソース          | 説明               |
| ---- | --- | ------------ | ---------------- |
| HTTP | 80  | `0.0.0.0/0`  | HTTP from CloudFront |
| SSH  | 22  | **自分のIP/32** | SSH from operator IP          |

> CloudFront のIP制限は **後で**
> Phase1ではまず疎通優先でOK

#### アウトバウンド

* 全許可（デフォルト）

👉 セキュリティグループを作成

---

### ⑤-2 RDS 用 SG

**EC2 → セキュリティグループ → セキュリティグループを作成**

| 項目  | 値                     |
| --- | --------------------- |
| セキュリティグループ名 | butterfly-room-rds-sg |
| 説明 | RDS PostgreSQL for butterfly-room. Access allowed only from EC2 SG. |
| VPC | butterfly-room-vpc    |

#### インバウンドルール

| タイプ        | ポート  | ソース                   |
| ---------- | ---- | --------------------- |
| PostgreSQL | 5432 | butterfly-room-ec2-sgの`sg-08bc1934afbbacb89`等 |

👉 セキュリティグループを作成

* **CIDR指定しないのが重要**
* EC2 SG からの通信のみ許可

#### アウトバウンド

* 全許可（デフォルト）
