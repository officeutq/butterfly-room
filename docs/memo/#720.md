## 実装方針

### 1. Controller

今は `@user` を取るだけなので、詳細ページで使う関連データを持たせます。
特に `booths/_card` は `booth_favorited` と `store_favorited` が必要なので、その分も用意します。

### 2. View

今の `users/show.html.erb` は「プロフィール単体ページ」なので、これを

* hero: avatar
* header: 名前 + role
* info: role / 関連数
* description: bio
* related list:

  * cast → 所属ブース
  * store_admin → 所属店舗

の形に寄せます。
構造としては `stores/show.html.erb` にかなり近いです。

## 差分案

### `app/controllers/users_controller.rb`

```ruby
# frozen_string_literal: true

class UsersController < ApplicationController
  before_action :authenticate_user!

  def show
    @user = User.where(deleted_at: nil).find(params[:id])

    @cast_booths = []
    @admin_stores = []
    @favorite_booth_ids = Set.new
    @favorite_store_ids = Set.new

    if @user.cast?
      @cast_booths =
        @user
          .cast_booths
          .active
          .includes(
            :store,
            :current_stream_session,
            { booth_casts: :cast_user },
            { thumbnail_image_attachment: :blob }
          )
          .order(:id)

      booth_ids = @cast_booths.map(&:id)
      store_ids = @cast_booths.map(&:store_id).uniq

      @favorite_booth_ids =
        current_user.favorite_booths.where(booth_id: booth_ids).pluck(:booth_id).to_set

      @favorite_store_ids =
        current_user.favorite_stores.where(store_id: store_ids).pluck(:store_id).to_set
    end

    if @user.store_admin?
      @admin_stores =
        @user
          .store_memberships
          .where(membership_role: :admin)
          .includes(:store)
          .map(&:store)

      store_ids = @admin_stores.map(&:id)

      @favorite_store_ids =
        @favorite_store_ids.merge(
          current_user.favorite_stores.where(store_id: store_ids).pluck(:store_id).to_set
        )
    end
  end
end
```

## 補足

`users/show.html.erb` で使っている CSS クラスはまだ無いはずなので、見た目は一旦素のままでも進められます。
#720 の時点では、まず構造を booth / store に揃えるのを優先で大丈夫です。

もし見た目もある程度寄せるなら、次のどちらかです。

* `store-show-*` にかなり近いクラスへ寄せて流用する
* `user-show-*` を新設して後で `_user_show.scss` を作る

初手は **後者** が安全です。store / booth の既存見た目を壊しにくいです。
