## 実装方針

今回の不具合は、`presence_summary` や `presence_poll_controller.js` 自体ではなく、**cast/live 画面から `presence-poll` の接続先 DOM が消えたこと**が本体です。
そのため、最小差分で安全に直すなら次の構成がよいです。

* `presence-poll` の root は `app/views/cast/booths/live.html.erb` に置く
* `countTarget` もその配下に置く
* `turbo-frame#stream_meta_display` はそのまま使う
* ただし **viewer_count の表示ノードだけは Turbo Frame の外側** に置く
* 見た目は 1 つの meta panel に見えるようにする

これなら、`_reloadMetaDisplay()` により `stream_meta_display` が差し替わっても、`presence-poll` の接続は維持され、同時接続者数更新が止まりません。

---

## 変更対象

## この実装で満たせること

* cast/live 画面で同時接続者数が 다시表示される
* 表示位置は `live-overlay-meta-slot` 内に収まる
* metadata 更新で `stream_meta_display` が差し替わっても、`presence-poll` は生きたまま
* cast/live 側では `pingUrl` を渡していないので、presence への書き込みはしない
* viewer 側には触れていないので既存挙動を壊しにくい

## 確認ポイント

実装後は次を見れば十分です。

1. cast/live 画面で meta panel に人数が出る
2. 数十秒待つか viewer を出入りして、人数が更新される
3. 配信設定の更新などで `stream_meta_display` が差し替わった後も、人数更新が継続する
4. cast/live 側から `ping` が飛んでいないことを Network で確認する

できます。
その形のほうがむしろ安全で、**右に人数の固定寄り領域、左に meta の残り全部** という構成にしやすいです。

やることは次の 2 点です。

* `live-meta-panel` の中を **2カラム化**
* 左カラムに `turbo-frame`、右カラムに `live-meta-viewer-count`

こうすれば、`countTarget` は引き続き Turbo Frame 外に置いたままなので安全です。

置き換え後の ERB はこれでいけます。

```erb
        <div class="live-overlay-meta-slot">
          <div
            class="live-meta-panel d-none"
            data-controller="presence-poll"
            data-ivs-publisher-target="metaPanel"
            data-presence-poll-summary-url-value="<%= presence_summary_stream_session_path(@stream_session) %>"
            data-presence-poll-interval-ms-value="20000"
          >
            <div class="live-meta-layout">
              <div class="live-meta-main">
                <turbo-frame id="stream_meta_display">
                  <%= render "cast/booths/stream_meta_display",
                            booth: @booth,
                            stream_session: @stream_session %>
                </turbo-frame>
              </div>

              <div class="live-meta-side">
                <div class="live-meta-viewer-count">
                  <i class="bi bi-people-fill" aria-hidden="true"></i>
                  <span data-presence-poll-target="count">-</span>
                </div>
              </div>
            </div>
          </div>
        </div>
```

SCSS は `live meta panel` セクションにこれを足せば大丈夫です。

```scss
.live-meta-layout {
  display: grid;
  grid-template-columns: minmax(0, 1fr) max-content;
  align-items: start;
  gap: 10px;
  min-width: 0;
}

.live-meta-main {
  min-width: 0;
}

.live-meta-main turbo-frame {
  display: block;
  min-width: 0;
}

.live-meta-side {
  flex: 0 0 auto;
}

.live-meta-viewer-count {
  display: inline-flex;
  align-items: center;
  gap: 6px;
  white-space: nowrap;

  font-size: 0.875rem;
  line-height: 1.2;
  color: rgba(255, 255, 255, 0.88);
}
```

この構成の挙動はこうなります。

* 左側は `minmax(0, 1fr)` なので残り全部を使う
* 右側は `max-content` なので、人数表示に必要な最小幅だけ使う
* Turbo Frame 差し替えは左側だけ
* `countTarget` は右側で維持されるので更新が止まりにくい

注意点としては、`_stream_meta_display.html.erb` 側を「タイトルと説明だけ」にしておくのが前提です。
つまり、前に出したこの形のままで相性がいいです。

```erb

<% title_display = stream_session.title.presence || booth.name %>

<div class="live-meta-main-row">
  <div class="live-meta-title"><%= title_display %></div>
</div>

<% if stream_session.description.present? %>
  <div class="live-meta-description">
    <%= simple_format(stream_session.description) %>
  </div>
<% end %>
```

これでかなりきれいに収まります。
