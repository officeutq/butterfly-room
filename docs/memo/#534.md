では、方針Bでそのまま実装します。
公開は `drink_icon` のみ、内部で path / label 解決をまとめ、`_card.html.erb` と `_form.html.erb` の両方で再利用する形です。

---

## 実装方針

* `app/helpers/drink_items_helper.rb` を新規作成
* 公開メソッドは `drink_icon(drink_item, size: 40, klass: "")`
* 内部補助メソッド

  * `drink_item_icon_path(drink_item)`
  * `drink_item_icon_label(drink_item)`
  * `drink_item_icon_option(drink_item)`
  * `drink_icon_option_by_key(icon_key)`
* `admin/drink_items/_card.html.erb`

  * 直接 `image_tag` を `drink_icon(...)` に置換
* `admin/drink_items/_form.html.erb`

  * 候補一覧の `image_tag` も、内部補助メソッド経由で path / label を参照
* CSS はそのままでOK

  * `drink-item-card-icon-image` は `klass` で渡す
  * helper側の inline style がサイズと `object-fit` を保証

## 1. `app/helpers/drink_items_helper.rb`

新規作成してください。

```rb
# frozen_string_literal: true

module DrinkItemsHelper
  def drink_icon(drink_item, size: 40, klass: "")
    image_tag(
      drink_item_icon_path(drink_item),
      alt: drink_item_icon_label(drink_item),
      title: drink_item_icon_label(drink_item),
      class: class_names("rounded-circle", klass),
      style: "width: #{size}px; height: #{size}px; object-fit: cover;"
    )
  end

  private

  def drink_item_icon_path(drink_item)
    drink_item_icon_option(drink_item)[:path]
  end

  def drink_item_icon_label(drink_item)
    drink_item_icon_option(drink_item)[:label]
  end

  def drink_item_icon_option(drink_item)
    drink_icon_option_by_key(drink_item&.icon_key)
  end

  def drink_icon_option_by_key(icon_key)
    return { label: "未設定", path: "drink_icons/drink_unset.jpg" } if icon_key.blank?

    DrinkItem::ICON_OPTIONS.fetch(icon_key, { label: "未設定", path: "drink_icons/drink_unset.jpg" })
  end
end
```

## 補足

今回の `_form.html.erb` は、公開helperを増やさずに内部補助メソッドを再利用するため、`send(:drink_icon_option_by_key, ...)` を使っています。

これは動きますが、見た目として少し気になるなら、次のどちらかにしてもよいです。

* helperの内部補助メソッドを private のまま維持して、今回のままいく
* `drink_icon_option_by_key` だけ public にして、form から素直に呼ぶ

ただ、今回の要件は
**「外部公開は `drink_icon` のみ」**
なので、まずはこの形が仕様に一番忠実です。
