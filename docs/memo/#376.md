以下で進めれば大丈夫です。
見た目は変えず、責務だけを 3 系統に分離する形にしています。

## 実装方針

* `application.html.erb`

  * 共通 `header` を partial 化
  * `default / cast-live / viewer` の main 分岐だけ担当
* `_default_main.html.erb`

  * 現在の通常 main + 共通 footer を移植
* `_cast_live_main.html.erb`

  * `yield` のみ
* `_viewer_main.html.erb`

  * viewer 用 flash + 本文 + footer を持つ
* `_header.html.erb`

  * 現在の header をそのまま移植

## 補足

今回の `_viewer_main.html.erb` では、Issue本文どおり「viewer main に footer まで持たせる」を優先しています。
中央の「配信」ボタン領域は viewer では不要なので空にしています。
これで通常 footer と viewer footer の責務が分かれ、次の viewer レイアウト調整 Issue に進めやすくなります。

## 動作確認ポイント

* 通常ページで従来どおり header / main / footer が出る
* `cast/booths/:id/live` で layout 側 footer が二重表示されない
* `booths/:id` で viewer 用 main + footer に切り替わる
* flash が通常ページと viewer で表示される
* cast/live の flash は引き続き `live_flash_layer` 側で表示される
