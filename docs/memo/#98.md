## 4) アプリ配置（git clone）

**Deploy Key（SSH）で clone する（最短・安全）**

* 「このEC2だけがこのリポジトリを読める」方式です。

### 1) EC2でSSH鍵を作る

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
ssh-keygen -t ed25519 -C "butterfly-room-ec2" -f ~/.ssh/butterfly_room_deploy -N ""
```

### 2) 公開鍵を表示してコピー

```bash
cat ~/.ssh/butterfly_room_deploy.pub
```

### 3) GitHub（officeutq/butterfly-room）側で登録

GitHub リポジトリ → **Settings** → **Deploy keys** → **Add deploy key**

* Title: `ec2-butterfly-room`
* Key: さっきの公開鍵を貼る
* ✅ Allow write access：**OFF**（基本OFFでOK）

### 4) EC2側でこの鍵を使う設定

```bash
cat > ~/.ssh/config <<'EOF'
Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/butterfly_room_deploy
  IdentitiesOnly yes
EOF
chmod 600 ~/.ssh/config
```

### 5) 接続確認（初回は known_hosts 登録）

```bash
ssh -T git@github.com
```

「Hi <username>!」みたいに出ればOK（Deploy keyだと “This key is not associated with a user” 系の文言でもOKで、認証できてればOK）

### 6) clone

```bash
mkdir -p ~/apps
cd ~/apps
git clone git@github.com:officeutq/butterfly-room.git
cd butterfly-room
```

## 5) 本番ENVを置く

### (A) RDS 接続情報を用意

RDS 作成後に控えたもの：

* DB_HOST=`xxxxx.ap-northeast-1.rds.amazonaws.com`
* DB_NAME=butterfly_room_production
* DB_USER=butterfly_room
* DB_PASSWORD=Secrets Manager の値

#### `.env.production` の `DATABASE_URL` を直す

パスワード部分を **URLエンコード**します。

例：

* `*` → `%2A`
* `(` → `%28`
* `)` → `%29`
* `|` → `%7C`
* `@` → `%40`
* `:` → `%3A`
* `/` → `%2F`
* `#` → `%23`

### (B) `.env.production` を作る

```bash
cd ~/apps/butterfly-room

cat > .env.production <<'EOF'
RAILS_ENV=production
NODE_ENV=production
RAILS_LOG_TO_STDOUT=1

RAILS_MASTER_KEY=REPLACE_ME
SECRET_KEY_BASE=REPLACE_ME

DATABASE_URL=postgres://DB_USER:DB_PASSWORD@DB_HOST:5432/DB_NAME

AWS_REGION=ap-northeast-1
AWS_S3_BUCKET=your-bucket-name
EOF
```

> ✅ `SECRET_KEY_BASE` は後で rails で生成するので一旦空でもいいが、最終的に必須です。
### (C) `.env.production` をEC2で編集する方法

EC2上で：

```bash
cd ~/apps/butterfly-room
vi .env.production
```

※ これは「EC2が書き込み禁止」ではなく「リポジトリを汚さない」だけなのでOK。

## 6) ローカルで追加するファイル（そのまま使える版）

### `docker-compose.production.yml`

```yaml
services:
  app:
    build:
      context: .
      dockerfile: Dockerfile.production
    image: butterfly_room:prod
    container_name: butterfly_room-app
    working_dir: /app
    env_file:
      - ./.env.production
    ports:
      - "3000:3000"
    environment:
      RAILS_ENV: production
      NODE_ENV: production
    restart: unless-stopped
```

### `Dockerfile.production`

```dockerfile
FROM ruby:3.3-slim

ENV LANG=C.UTF-8 TZ=Asia/Tokyo \
    BUNDLE_PATH=/usr/local/bundle \
    BUNDLE_JOBS=4 BUNDLE_RETRY=3 \
    PATH="/usr/local/bundle/bin:${PATH}"

RUN apt-get update -qq && apt-get install -y --no-install-recommends \
    build-essential libpq-dev curl git nodejs npm \
    pkg-config libyaml-dev \
  && rm -rf /var/lib/apt/lists/*

WORKDIR /app

# 先に Gemfile だけコピーして bundle install をキャッシュさせる
COPY Gemfile Gemfile.lock ./
RUN gem update --system -N && gem install bundler -N
RUN bundle config set deployment 'true' \
 && bundle config set without 'development test' \
 && bundle install

# アプリ本体をコピー
COPY . .

# 本番起動（Railsが 0.0.0.0 でlisten）
CMD ["bash", "-lc", "bundle exec rails server -b 0.0.0.0 -p 3000"]
```

### `.gitignore` 追記

```gitignore
.env.production
```

## 5) 動作確認コマンド（EC2）

```bash
cd ~/apps/butterfly-room
docker compose -f docker-compose.production.yml --env-file .env.production build --no-cache
docker compose -f docker-compose.production.yml --env-file .env.production up -d
docker logs -f butterfly_room-app
```
* `SECRET_KEY_BASE` は本番で必須（空NG）
* もしまだなら作る：

  ```bash
  docker compose -f docker-compose.production.yml --env-file .env.production run --rm app bin/rails secret
  ```

  出た値を `.env.production` の `SECRET_KEY_BASE=` に貼る

### migrate：

```bash
docker compose -f docker-compose.production.yml --env-file .env.production exec app \
  bash -lc 'bundle exec rails db:migrate'
```
#### schema_migrations で確認

```bash
docker compose -f docker-compose.production.yml --env-file .env.production exec app \
  bin/rails db:migrate:status
```
#### RDS に直接効いているか（念のため）

```bash
docker compose -f docker-compose.production.yml --env-file .env.production exec app \
  bin/rails runner "puts ActiveRecord::Base.connection.tables"
```

テーブル名が配列で出れば **RDSにちゃんと書き込めてる**。
