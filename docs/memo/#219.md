## 変更コード（貼り付け用・差し替え）

**ファイル:** `app/views/layouts/application.html.erb`
フッターの「配信」部分（`<div> ... </div>`）を下記に置き換え。

```erb
          <div>
            <% if user_signed_in? && current_user.at_least?(:cast) %>
              <% stream_path =
                   if current_user.cast?
                     booth = layout_current_booth
                     if booth.present?
                       booth.offline? ? cast_booth_path(booth) : live_cast_booth_path(booth)
                     else
                       cast_booths_path
                     end
                   else
                     cast_booths_path
                   end %>

              <%= link_to "配信", stream_path, class: "btn btn-sm btn-primary" %>
            <% end %>
          </div>
```

### ねらい

* cast だけ “ワンクリック” を優先
* store_admin / system_admin は常にブース一覧（`cast_booths_path`）へ
* `layout_current_booth` が nil を返すケース（未選択/権限外/不整合）も自然に `cast_booths_path` へフォールバック

## PR本文（コピペ用）

**Title:** [Improve] フッター「配信」導線：castはワンクリック配信、admin系はブース一覧へ #219

**概要**

* フッター中央の「配信」ボタンの遷移先を、ロールと current_booth / booth.status で統一。
* cast は current_booth が解決できる場合にワンクリックで配信画面へ遷移できる体験を優先。
* store_admin / system_admin は常に cast ブース一覧へ誘導。

**変更点**

* `app/views/layouts/application.html.erb`

  * 「配信」ボタンのリンク先を以下で分岐

    * cast:

      * current_booth なし → `cast_booths_path`
      * booth.offline? → `cast_booth_path(booth)`
      * booth.standby/live/away → `live_cast_booth_path(booth)`
    * store_admin / system_admin: 常に `cast_booths_path`
  * 表示条件（`at_least?(:cast)`）は従来どおり維持

**仕様（確認）**

* cast は current_booth がある場合、offline は summary（show）、それ以外は live へ
* store_admin / system_admin は「配信」押下で cast ブース一覧へ
* current_booth が無い cast はブース一覧へ

**動作確認**

* [ ] cast（current_booth あり / offline）で「配信」→ summary へ
* [ ] cast（current_booth あり / standby|live|away）で「配信」→ live へ
* [ ] cast（current_booth なし）で「配信」→ cast ブース一覧へ
* [ ] store_admin / system_admin で「配信」→ cast ブース一覧へ
* [ ] 既存の Current整合 / BAN / archived の挙動に影響がないこと（参照は既存 helper を利用）

**Closes**

* Closes #219
