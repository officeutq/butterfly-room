## ① ALB 用の SG を新規作成

**EC2 → セキュリティグループ → セキュリティグループを作成**
* セキュリティグループ名：`butterfly-room-alb-sg`
* 説明：`ALB security group for CloudFront HTTP HTTPS`
* VPC：`vpc-05d7e03b48c391388 (butterfly-room-vpc)`
* インバウンドルール
  | タイプ | ソース |
  | --| -- |
  | HTTP | 0.0.0.0/0 |
  | HTTPS | 0.0.0.0/0 |

👉 セキュリティグループを作成
※ セキュリティグループ ID をメモ

## ② butterfly-room-ec2-sgの SG を編集
**EC2 → セキュリティグループ → butterfly-room-ec2-sg → インバウンドのルールを編集**
* `3000`を削除 → ルールを追加
  * タイプ：`カスタム TCP`
  * ポート範囲：`3000`
  * ソース `butterfly-room-alb-sg` を選択
  * 説明：`EC2 does not receive 3000 from anything other than ALB`
👉 ルールを保存

> ポイント：EC2の3000は **ALB SG をソースにする**のが綺麗。


## ③ ターゲットグループ（Target Group）を作成

**EC2 → ロードバランサー → ターゲットグループ → ターゲットグループの作成**

* ターゲットの種類: **インスタンス**
* ターゲットグループ名：`butterfly-room-app-tg`
* プロトコル: **HTTP**
* ポート: **3000**
* VPC: `vpc-05d7e03b48c391388 (butterfly-room-vpc)`
* プロトコルバージョン：HTTP1（HTTP/1.1）
* ヘルスチェック

  * プロトコル: HTTP
  * パス: **`/up`**（Rails標準のヘルスチェックがあるはず。無い場合は `/` でも暫定OK）
  * 成功コード: `200`

作成後、「ターゲットの登録」で

* 対象インスタンス（butterfly-room-app-1）を選択
* ポートは **3000**
* 次へ → ターゲットグループの作成


## ④ ALB を作成

**EC2 → ロードバランサー → ロードバランサーの作成 → Application Load Balancer**

* ロードバランサー名: `butterfly-room-alb`
* スキーム: **インターネット向け**
* IP アドレスタイプ: IPv4
* リスナー:

  * まずは **HTTP:80** だけでOK（HTTPSは後で証明書できてから）
* ネットワークマッピング:

  * VPC: `butterfly-room-vpc`
  * **パブリックサブネットを2つ**選ぶ（public-a と public-c みたいに別AZのもの）
* セキュリティグループ:

  * **`butterfly-room-alb-sg`** を選択（他は外す）
* ルーティング（ターゲットグループ）:

  * さっき作った `butterfly-room-tg`（仮名）を選択

作成。

## ⑤ すぐ動作確認

ALB の画面で **DNS名**（`xxxx.ap-northeast-1.elb.amazonaws.com` みたいなの）が出るので

* ブラウザで:
  `http://<ALBのDNS名>/`

さらに、ターゲットグループの「ターゲット」タブで

* **healthy** になってるか確認
