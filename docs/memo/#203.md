# 🔷 PHASE 1：ACM 証明書（2リージョン）

## ① CloudFront 用（必須・us-east-1）

リージョンを **us-east-1 (N. Virginia)** に切り替える。

ACM → Request certificate

* ドメイン名：`butterflyve.jp`
* 検証：DNS validation
* Route53 自動作成でOK

発行完了まで待つ（数分）

---

## ② ALB 用（ap-northeast-1）

ACM → Request certificate

* ドメイン名：`butterflyve.jp`
* DNS validation
* Route53 自動作成

発行完了まで待つ

---

# 🔷 PHASE 2：ALB に HTTPS 443 リスナー追加

EC2 → Load Balancers → 対象ALB

Listeners → Add listener

* Protocol：HTTPS
* Port：443
* Certificate：先ほどの ap-northeast-1 ACM
* Default action：既存 Target Group に forward

保存

---

## 🔥 Security Group確認（重要）

ALB の Security Group に：

* Inbound 443 → 0.0.0.0/0

を追加（なければ）

# 🔷 PHASE 3：CloudFront → ALB を HTTPS に変更

CloudFront → Distribution → Origins → Edit

変更：

* Origin protocol policy：**HTTPS only**
* HTTPS port：443
* Origin SSL protocols：TLSv1.2

保存 → デプロイ待ち（5〜10分）

# 🔷 PHASE 4：CloudFront に独自ドメイン追加

CloudFront → General → Edit

* Alternate domain names に：

  us-east-1 の ACM を選択

保存

デプロイ待ち

Route53 → butterflyve.jp Hosted Zone
