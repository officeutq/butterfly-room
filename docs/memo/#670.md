# ① EC2 の IAM ロール確認

### 手順

1. AWS コンソールにログイン
2. **EC2** を開く
3. 左メニュー → 「インスタンス」
4. 対象インスタンス（butterfly-room）をクリック
5. 下の詳細タブ → **セキュリティ**
6. 「IAM ロール」を確認

👉 例:

```
butterfly-room-ec2-role
```

---

## 次にやること

7. ロール名をクリック（IAMに飛ぶ）

---

# ② IAM ロールに SNS 権限を追加

## 確認

1. IAM 画面でロールを開く
2. 「許可」タブ
3. ポリシー一覧を見る

👉 `sns` が無ければ追加

---

## 追加方法

1. 「許可を追加」→「ポリシーをアタッチ」
2. 検索で

```
AmazonSNSFullAccess
```

👉 最初はこれでOK（後で絞れる）

---

### ちゃんとやるなら（推奨）

カスタムポリシー👇

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "sns:Publish"
      ],
      "Resource": "*"
    }
  ]
}
```

---

# ③ SNS SMS 設定確認

## 手順

1. AWS コンソール
2. **Amazon SNS**
3. 左メニュー → **Text messaging (SMS)**

---

## 確認ポイント

### ① SMS sandbox

表示されるはず👇

* Sandbox mode → ON / OFF

👉 ONなら本番不可

---

### ② Monthly spend limit

例:

```
1 USD
```

👉 OK（後で上げる）

---

### ③ Default message type

```
Transactional
```

👉 必ずこれ

---

# ④ Production access 申請

## 手順

1. SNS → SMS 画面
2. 上の方にある

👉 「Request production access」

をクリック

---

## 入力内容（そのまま使ってOK）

```text
Use case:
We use Amazon SNS to send one-time passwords (OTP) for user authentication.

Message type:
Transactional

Target countries:
Japan

Estimated monthly volume:
Initially less than 1,000 messages per month, may grow up to 5,000.

Opt-in:
Users voluntarily enter their phone number and request a verification code.

Opt-out:
Not applicable. Messages are only sent when users request authentication.

Sample message:
Your Butterflyve verification code is 123456
```

---

## 送信後

* 数時間〜1日で承認されることが多い

# ⑤ 日本向け仕様の確認（重要）

SNS SMSは国ごとに仕様違います。

## 日本の場合

* Sender ID → 基本使えない
* 電話番号 → AWSの番号で送信
* OTP用途 → 問題なし

👉 つまりそのままでOK

---
