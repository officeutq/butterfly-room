## 実装プラン（#317）

優先順で、Done条件に沿って最小変更で進めます。

1. **HTMLの骨格を正す**

* いまの `live.html.erb` は outer `<div ...>` と `#live_stage` と footer の閉じタグが分かりづらいので、まず「外枠（controllers）→ stage → footer → 閉じ」を明確にする。

2. **container を live から撤去**

* `flash_inner` を含む `container mt-3` を撤去（ただし `#flash_inner` 自体は残す）
* `container my-4` を撤去（代わりに “Stage内の通常ブロック” にする）

3. **card/card-header/card-body を撤去**

* 「配信（Real-Time Stage）」ブロックは card なしで、最小の wrapper（`div`）にする
* `data-ivs-publisher-target="state"` と `summaryBtn` は温存（配置だけ変える）

4. **aspect-ratio: 16/9 を撤去**

* `video` / `canvas` のインライン style から `aspect-ratio: 16 / 9;` を削除
* `background: #000;` は残す（黒帯仕様の下地として有用）

5. **video / canvas を Stage直下へ寄せる**

* 本当に「#live_stage 直下 ＝ video/canvasだけ」までやると、コメント/ドリンク等の“後続overlay対象”が置けなくなるので、
  #317 では **「配信映像ブロック（video/canvas/error/操作）を Stage の先頭（直下に近い位置）に出す」**までに留める。
  （コメント/ドリンクは別Issueで overlay に移す）

6. **動作確認（手順）**

* standbyで開く→プレビューonlyが出る（tokenUrlあり前提）
* 配信開始→join→live化→ボタンが切り替わる
* 離席→canvas表示/音声トグル→復帰→preview表示
* 配信終了→finish→リロード→サマリーへ遷移
* pending_drink_orders / comments の Turbo 更新が死んでない
