貼ってくれた範囲だけで **原因はほぼ確定**です。

* `config.force_ssl` が **OFF**
* `config.assume_ssl` も **OFF**
* その結果、CloudFront(HTTPS) →（どこかでHTTP扱い）→ Rails が `request.base_url` を `http://...` と判断
* Rails の CSRF Origin チェックで **Origin=https と base_url=http が不一致**になって 422

---

## まず入れる修正（最短で直す）

### 1) `production.rb` で SSL を「前段終端でも https 扱い」にする

`config/environments/production.rb` をこうします（差分イメージ）：

```diff
  # Force all access to the app over SSL, use Strict-Transport-Security, and use secure cookies.
- # config.force_ssl = true
+ config.force_ssl = true

  # Assume all access to the app is happening through a SSL-terminating reverse proxy.
- # config.assume_ssl = true
+ config.assume_ssl = true

+ # Skip http-to-https redirect for the default health check endpoint.
+ config.ssl_options = { redirect: { exclude: ->(request) { request.path == "/up" } } }

+ # Allow CloudFront host (暫定: いまの distribution を許可)
+ config.hosts << "d1qeq5amxvhfv3.cloudfront.net"
```

これで Rails 側の scheme 認識が `https` に寄り、`request.base_url` が `https://...` になって **CSRFが通る**ようになります。

> `force_ssl` と `assume_ssl` の両方を ON にするのは「CloudFront→ALB が HTTP で来ても Rails は HTTPS として扱う」ための保険です。
> （CloudFront/ALBの設定がまだ揺れてても、まずログインを復旧させる狙い）

## ついでに直すべき（放置すると別事故になるやつ）

### 2) `action_mailer.default_url_options` の host を暫定でも実ドメインに

今 `example.com` のままだと、Devise のメール（パスワード再設定等）のリンクが壊れます。

```diff
- config.action_mailer.default_url_options = { host: "example.com" }
+ config.action_mailer.default_url_options = { host: "d1qeq5amxvhfv3.cloudfront.net", protocol: "https" }
```

（独自ドメインを入れたら、あとでそこに変更）

## インフラ側で「正しく」直す（推奨・再発防止）

アプリ側だけでも直りますが、根っこはここ：

**CloudFront→ALB が HTTP で入ってきて、Rails が http と認識している**可能性が高いです。

なので CloudFront の Origin 設定で、可能なら

* **Origin Protocol Policy: HTTPS Only**（または Match Viewer）

に寄せるのが王道です。これができると `assume_ssl` への依存も減ります。

## 動作確認チェック（これだけ見ればOK）

修正後に production で

1. `https://d1qeq5amxvhfv3.cloudfront.net/users/sign_in` でログイン
2. Railsログで、エラー行が消えていること
3. 可能ならログに一時的に（短期間だけ）以下を出すと確証が取れます

   * `request.base_url`
   * `request.ssl?`
   * `request.headers["X-Forwarded-Proto"]`

（ただ、まずは `force_ssl + assume_ssl` を入れれば大体一発で直ります）
