## 実装方針（#293）

### ねらい

* **`cast/booths#live` のときだけ**、`application.html.erb` から **共通 `<main>` と共通 `<footer>` を出さない**
* 代わりに `cast/booths/live.html.erb` 側で **Main相当（flash_inner含む）＋ Footer相当（共通と同等見た目）**を描画
* Header と `<turbo-frame id="modal">` は **従来どおり layout に残す**（共通要素維持）

### live判定（固定）

* `controller_path == "cast/booths"` かつ `action_name == "live"`

## 動作確認チェック（Issueの「動作確認」に対応）

1. **cast live**（`/cast/booths/:id/live`）で

* footer が **二重表示にならない**（共通footerが消えて、ページ側footerのみ）
* footer 固定で本文が被らない（main が `pb-5` で余白確保）
* `flash_inner` が存在し、Turbo Stream append/replace が機能する（BAN通知・配信終了通知など）

2. **live以外**（例：root / booths show / dashboard 等）で

* 従来どおり layout の `<main>` と共通footerが表示される（見た目・導線が変わらない）

3. **グローバルモーダル**

* 既存どおり `<turbo-frame id="modal">` が機能する（enterのモーダル等）
