ブランチ名候補です。

* `improve/362-live-flash-layer`
* `feature/362-live-flash-overlay-layer`
* `improve/362-dismissible-live-flash`
* `feature/362-live-notification-layer`

おすすめは `improve/362-live-flash-layer` です。

実装方針です。

* `live.html.erb`

  * `#flash_inner` を `#live_stage` の外から中へ移動
  * `#live_overlay` とは別に `#live_flash_layer` を追加
* `_flash_message.html.erb`

  * Bootstrap の dismissible alert に変更
* `application.scss`

  * `#live_flash_layer` と live専用 flash 見た目を追加
  * レイヤー自体は `pointer-events: none`
  * alert 本体だけ `pointer-events: auto`
* controller 側

  * `flash_inner` への append は現状のままでOK

はい、その通りです。
**redirect 後の通常 flash に × を付けたいだけなら、まさにそこを直せばOK** です。

つまり今の

```erb
<% if notice.present? %>
  <div class="alert alert-success" role="alert"><%= notice %></div>
<% end %>

<% if alert.present? %>
  <div class="alert alert-danger" role="alert"><%= alert %></div>
<% end %>
```

を、dismissible な Bootstrap alert にすれば、`プロフィールを更新しました` にも × が出ます。

ただ、さらに一歩進めるなら、**そこを直接書き換えるより `shared/_flash_message.html.erb` を呼ぶ形に寄せたほうが統一的**です。
理由は、live の Turbo Stream append でも同じ partial を使っているからです。

つまりおすすめはこれです。

```erb
<% if notice.present? %>
  <%= render "shared/flash_message", level: "success", message: notice %>
<% end %>

<% if alert.present? %>
  <%= render "shared/flash_message", level: "danger", message: alert %>
<% end %>
```

こうすると、

* redirect 後の notice / alert
* Turbo Stream で append される flash

が同じ見た目・同じ挙動になります。

逆に、その場で直書きで × を付けるだけでも動きますが、そうすると

* layout 側の flash
* live 側の flash
* shared partial の flash

で記述が分散しやすいです。

なので結論はこうです。

* **今のその場所を直せば × は出る**
* でも、**共通 partial を呼ぶ形にしたほうがきれい**

必要なら次に、`application.html.erb` と `live.html.erb` の置換箇所だけをコピペ用で出します。
