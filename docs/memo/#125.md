## 🔧 実装①：presence_summary を落ちにくくする（本番安定化）

今の `presence_summary` は、どこかで例外が出ると 500 になります。

これは

* viewer 側 polling が止まる
* joinable イベントが来なくなる

という本番事故の原因になります。

### 修正後コード（安全版）

`app\controllers\stream_sessions_controller.rb`

```rb
def presence_summary
  viewer_count =
    begin
      Presences::SummaryService.new(
        stream_session: @stream_session,
        threshold_seconds: 45
      ).call!
    rescue => e
      Rails.logger.warn("[presence_summary] viewer_count failed: #{e.class} #{e.message}")
      0
    end

  joinable =
    begin
      Ivs::CreateParticipantTokenService.new(
        stream_session: @stream_session,
        actor: current_user,
        role: Ivs::CreateParticipantTokenService::ROLE_VIEWER
      ).joinable?
    rescue => e
      Rails.logger.warn("[presence_summary] joinable failed: #{e.class} #{e.message}")
      false
    end

  render json: { viewer_count: viewer_count, joinable: joinable }
end
```

### これで何が保証されるか

* SummaryService が壊れても viewer UI は止まらない
* joinable 判定が壊れても false にフォールバック
* 本番でも polling が必ず返る
* #125 Done条件「本番・別端末でも成立」が担保される
