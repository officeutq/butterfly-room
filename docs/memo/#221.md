## 実装プラン（最小差分）

1. `app/views/layouts/application.html.erb` のヘッダー内、`store:` / `booth:` の表示部分を置き換える
2. role 分岐は次のとおり

   * cast：`booth:` だけリンク（`cast_booths_path`）。`store:` はリンクなし
   * store_admin / system_admin：`store:` → `admin_stores_path`、`booth:` → `admin_booths_path`
   * customer：そもそも表示しない（現状どおり `unless current_user.customer?` の外側で担保）
3. Current が未選択の場合も、表示している「未選択」を **同じ遷移先へのリンク**にする（文言はそのまま）
4. 見た目は最小：`link-secondary link-underline-opacity-0 link-underline-opacity-100-hover` を付けて、テキストっぽく見せつつ hover でリンク感が出る程度にする（Bootstrap標準の範囲）

## 変更コード（貼り替え対象：`app/views/layouts/application.html.erb` の一部）

**置換対象ブロック**：`<% unless current_user.customer? %>` 〜 `</span>`（booth の span まで）を、下記で丸ごと置き換え。

```erb
              <% unless current_user.customer? %>
                <% store = layout_current_store %>
                <span class="text-muted small">
                  store:
                  <% if current_user.at_least?(:store_admin) %>
                    <%= link_to admin_stores_path,
                          class: "link-secondary link-underline-opacity-0 link-underline-opacity-100-hover" do %>
                      <% if store.present? %>
                        <strong><%= store.name.presence || "(no name)" %></strong>
                      <% else %>
                        <span class="text-danger">未選択</span>
                      <% end %>
                    <% end %>
                  <% else %>
                    <%# cast: store は表示のみ（クリック不可） %>
                    <% if store.present? %>
                      <strong><%= store.name.presence || "(no name)" %></strong>
                    <% else %>
                      <span class="text-danger">未選択</span>
                    <% end %>
                  <% end %>
                </span>

                <% booth = layout_current_booth %>
                <span class="text-muted small">
                  booth:
                  <% if current_user.cast? %>
                    <%= link_to cast_booths_path,
                          class: "link-secondary link-underline-opacity-0 link-underline-opacity-100-hover" do %>
                      <% if booth.present? %>
                        <strong><%= booth.name.presence || "(no name)" %></strong>
                      <% else %>
                        <span class="text-danger">未選択</span>
                      <% end %>
                    <% end %>
                  <% elsif current_user.at_least?(:store_admin) %>
                    <%= link_to admin_booths_path,
                          class: "link-secondary link-underline-opacity-0 link-underline-opacity-100-hover" do %>
                      <% if booth.present? %>
                        <strong><%= booth.name.presence || "(no name)" %></strong>
                      <% else %>
                        <span class="text-danger">未選択</span>
                      <% end %>
                    <% end %>
                  <% else %>
                    <%# 念のため（到達しない想定）：customer は外で除外済み %>
                    <% if booth.present? %>
                      <strong><%= booth.name.presence || "(no name)" %></strong>
                    <% else %>
                      <span class="text-danger">未選択</span>
                    <% end %>
                  <% end %>
                </span>
              <% end %>
```

### 意図の補足（実装上のポイント）

* `current_user.at_least?(:store_admin)` は store_admin と system_admin をまとめてカバーします。
* cast は `store:` をあえてリンクにしません（仕様確定）。
* `未選択` のままでもリンクとして機能します（Done条件）。

**Title**
`[Improve] ヘッダーのstore/boothを切替導線にする（Current操作可能化） #221`

**Body**

* 目的

* 仕様メモ

  * customer は store/booth を表示しない（既存方針のまま）。
  * Current整合ロジック / session補正の挙動は変更なし。

* 動作確認

  * cast：booth（選択/未選択）が `/cast/booths` に遷移すること
  * store_admin / system_admin：store が `/admin/stores`、booth が `/admin/booths` に遷移すること（未選択でもOK）
  * customer：ヘッダーに store/booth が表示されないこと

Closes #221
