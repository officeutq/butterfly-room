## 実装方針

今回の実装は、**専用の edit アクションは増やさず**、`index` に `editing_id` を渡してカード単位で表示/編集を切り替える形にしています。

* 「編集」押下 → `admin_drink_items_path(editing_id: item.id)` に遷移
* そのカードだけフォーム表示
* 保存成功 → `index` へ戻るので閲覧状態に戻る
* バリデーションエラー → そのカードの編集状態を維持
* キャンセル → `admin_drink_items_path` に戻る
* 有効/無効は既存 `update` を使ったトグルのまま
* `destroy` は使わなくなるが、今回は routes/controller からは消さず放置でOK

# 4. `app/controllers/admin/drink_items_controller.rb`

```ruby
# frozen_string_literal: true

module Admin
  class DrinkItemsController < BaseController
    before_action :require_current_store!
    before_action :set_drink_item, only: %i[update destroy]

    def index
      @drink_item = current_store.drink_items.new(enabled: true, position: default_position)
      @drink_items = current_store.drink_items.ordered
      @editing_drink_item_id = params[:editing_id].to_i if params[:editing_id].present?
    end

    def create
      @drink_item = current_store.drink_items.new(drink_item_params)

      if @drink_item.save
        redirect_to admin_drink_items_path, notice: "作成しました"
      else
        @drink_items = current_store.drink_items.ordered
        @editing_drink_item_id = nil
        render :index, status: :unprocessable_entity
      end
    end

    def update
      if @drink_item.update(drink_item_params)
        redirect_to admin_drink_items_path, notice: "更新しました"
      else
        @drink_items = current_store.drink_items.ordered
        @editing_drink_item_id = @drink_item.id
        @drink_item = current_store.drink_items.new(enabled: true, position: default_position)
        render :index, status: :unprocessable_entity
      end
    end

    def destroy
      @drink_item.update!(enabled: false) # 論理削除
      redirect_to admin_drink_items_path, notice: "無効にしました"
    end

    private

    def set_drink_item
      @drink_item = current_store.drink_items.find(params[:id])
    end

    def drink_item_params
      params.require(:drink_item).permit(:name, :price_points, :position, :enabled, :icon_key)
    end

    def default_position
      last = current_store.drink_items.maximum(:position)
      last.present? ? last + 1 : 0
    end
  end
end
```

## 3. 今回の修正で変わること

* `grid-template-columns: repeat(auto-fit, minmax(0, 1fr))` にして、カードが内容幅に引っ張られにくくなります
* スマホ幅では `.drink-item-card-main` を `flex-direction: column;` にして、**アイコン上・内容下** に切り替わります
* 行もスマホ時は縦積みにして、ラベル幅固定が横はみ出しの原因にならないようにしています
* icon toggle のCSSは全部 `_drink_item.scss` に集約されます

必要なら次に、コミットメッセージとPR文もそのまま出します。

はい、**普通に Bootstrap ベースでできます**。
今回みたいに「カード幅が内容に引っ張られてはみ出す」系は、**独自 grid / 独自 flex を減らして Bootstrap の row / col に寄せたほうが安定**します。

今回のはたぶん、

* `.drink-items-list` の独自 grid
* `.drink-item-card-main` の独自 flex
* `.drink-item-card-row` の横並び
* `input-group` や icon toggle の横スクロール要素

あたりが組み合わさって、どこかが最小幅を押し広げています。

なので方針を変えて、**「一覧もカード中身も Bootstrap の縦積み中心」にする**のがおすすめです。

## 2. カード側の「有効/無効」をチェックボックス見た目に寄せる

これもできます。
ただし、**本物のチェックボックス1個だけで即 submit** にする場合は少し工夫が必要です。

---

## やり方は2つあります

### A. 見た目だけチェックボックス風にする

おすすめです。
実装が簡単で壊れにくいです。

つまり、実体は今まで通り `submit` ボタンだけど、見た目を

* checkedっぽい
* form-checkっぽい

に寄せる方法です。

これはCSSだけでもできます。

---

### B. 本当に checkbox にして、変更時に自動submit

これも可能です。
ただし、

* hidden field
* onchange submit
* update時に他項目を壊さない

を考える必要があるので、少しだけ複雑です。

今の controller は `update` で `name`, `price_points`, `position`, `icon_key`, `enabled` をまとめて受ける設計なので、
**カード側トグルでも他項目を hidden で維持する必要がある**のは今と同じです。

---

## おすすめ

今回は **A寄り** か、**Bでも hidden維持つき** が安全です。

でもユーザーの希望的には

> 見た目をフォーム側に寄せてチェックボックスにしたい

なので、**Bで本物チェックボックス**でも十分いけます。

---

## 3. 実際の書き換え例（カード側）

`_card.html.erb` のアクション部分を、今の

```erb
<div class="d-flex gap-2 flex-wrap mt-3">
  <%= form_with model: [:admin, drink_item], method: :patch, class: "d-inline" do |f| %>
    <%= f.hidden_field :name, value: drink_item.name %>
    <%= f.hidden_field :price_points, value: drink_item.price_points %>
    <%= f.hidden_field :position, value: drink_item.position %>
    <%= f.hidden_field :icon_key, value: drink_item.icon_key %>
    <%= f.hidden_field :enabled, value: (!drink_item.enabled).to_s %>
    <%= f.submit(drink_item.enabled ? "有効" : "無効",
                 class: "btn btn-sm #{drink_item.enabled ? "btn-success" : "btn-outline-secondary"}") %>
  <% end %>

  <%= link_to "編集",
              admin_drink_items_path(editing_id: drink_item.id),
              class: "btn btn-sm btn-outline-primary" %>
</div>
```

から、こう変えられます。

```erb
<div class="d-flex gap-2 flex-wrap mt-3 align-items-center">
  <%= form_with model: [:admin, drink_item], method: :patch, class: "d-inline-block" do |f| %>
    <%= f.hidden_field :name, value: drink_item.name %>
    <%= f.hidden_field :price_points, value: drink_item.price_points %>
    <%= f.hidden_field :position, value: drink_item.position %>
    <%= f.hidden_field :icon_key, value: drink_item.icon_key %>
    <%= f.hidden_field :enabled, value: "0" %>

    <div class="form-check mb-0">
      <%= f.check_box :enabled,
                      {
                        class: "form-check-input",
                        checked: drink_item.enabled,
                        onchange: "this.form.requestSubmit()"
                      },
                      "1",
                      "0" %>
      <%= f.label :enabled, "有効", class: "form-check-label" %>
    </div>
  <% end %>

  <%= link_to "編集",
              admin_drink_items_path(editing_id: drink_item.id),
              class: "btn btn-sm btn-outline-primary" %>
</div>
```

---

## 4. この方式で大丈夫な理由

Rails の checkbox は、

* hiddenで `0`
* checkedなら `1`

を送るので、`enabled` の true/false 切り替えに向いています。

さらに今の `drink_item_params` は `:enabled` を permit しているので、そのままでOKです。

---

## 5. 注意点

1つだけあります。

### `f.hidden_field :enabled, value: "0"` は不要

`check_box` 自体が hidden を出すので、**別で enabled hidden を置かない**ほうがいいです。

つまりカード側はこうです。

```erb
<%= f.hidden_field :name, value: drink_item.name %>
<%= f.hidden_field :price_points, value: drink_item.price_points %>
<%= f.hidden_field :position, value: drink_item.position %>
<%= f.hidden_field :icon_key, value: drink_item.icon_key %>
<%= f.check_box ... %>
```

で十分です。

### できます

* カード側の「有効/無効」もチェックボックスUIにできる

### おすすめ実装

* フォーム側は `row + col` にする
* カード側は `check_box + onchange: this.form.requestSubmit()` にする
