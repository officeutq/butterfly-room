# DB関連

## 方針

* 本番DBは **同名で再作成してリセット** する
* Rails本体は `db:schema:load` で復元する
* Solid系（Cable / Queue）は **EC2上で install → migrate で復元する**
* マスタ系データは事前に seed 化しておき、最後に投入する

---

## 手順

### 1. DBバックアップ

#### フルダンプ（推奨）

```bash
# PostgreSQL client が無い場合
sudo dnf install -y postgresql15

pg_dump "postgres://butterfly_room:MasterPassword1234@butterfly-room-postgres.cviquuwo4fu6.ap-northeast-1.rds.amazonaws.com:5432/butterfly_room_production?sslmode=require" \
  --format=custom \
  --file=backup_$(date +%Y%m%d_%H%M%S).dump
```

#### リストア

まず、バックアップファイルが存在することを確認する。

```bash
ls -lh *.dump
```

必要に応じて、バックアップファイルとして読み取れることも確認する。

```bash
pg_restore -l backup_*.dump | head
```

その後、実際のファイル名を指定してリストアする。

```bash
pg_restore \
  --dbname="postgres://butterfly_room:MasterPassword1234@butterfly-room-postgres.cviquuwo4fu6.ap-northeast-1.rds.amazonaws.com:5432/butterfly_room_production?sslmode=require" \
  --clean \
  --no-owner \
  backup_YYYYMMDD_HHMMSS.dump
```

例:

```bash
pg_restore \
  --dbname="postgres://butterfly_room:MasterPassword1234@butterfly-room-postgres.cviquuwo4fu6.ap-northeast-1.rds.amazonaws.com:5432/butterfly_room_production?sslmode=require" \
  --clean \
  --no-owner \
  backup_20260423_123456.dump
```

### 補足

* `backup_xxxxx.dump` はダミー名ではなく、**実際に作成されたファイル名** に置き換えること
* `--clean` は既存オブジェクトを削除しながら復元するため、対象DBを誤らないこと
* 本番DBへ直接実行する場合は、接続先を十分確認してから実行すること

#### スキーマ確認用（任意）

```bash
pg_dump "postgres://butterfly_room:MasterPassword1234@butterfly-room-postgres.cviquuwo4fu6.ap-northeast-1.rds.amazonaws.com:5432/butterfly_room_production?sslmode=require" \
  --schema-only \
  > schema_$(date +%Y%m%d_%H%M%S).sql
```

### 2. DBリセット

```bash

psql "postgres://.../postgres?sslmode=require" -c "
SELECT pg_terminate_backend(pid)
FROM pg_stat_activity
WHERE datname = 'butterfly_room_production'
  AND pid <> pg_backend_pid();
"

# DB削除
psql "postgres://.../postgres?sslmode=require" -c "DROP DATABASE IF EXISTS butterfly_room_production;"

# DB再作成
psql "postgres://.../postgres?sslmode=require" -c "CREATE DATABASE butterfly_room_production OWNER butterfly_room;"
```

# マスタデータ退避

## 方針

* マスタは YAML として退避し、seed で復元する
* 対象は **一覧（定義リスト）で管理** する

---

## 退避対象

* `referral_codes`
* `effects`

追加は `EXPORT_TARGETS` に定義を追加する

---

## 退避スクリプト

`script/export_master_data.rb`

```ruby
require "yaml"
require "fileutils"

output_dir = Rails.root.join("db", "seed_exports")
FileUtils.mkdir_p(output_dir)

EXPORT_TARGETS = [
  {
    name: :referral_codes,
    model: ReferralCode,
    order: :id,
    columns: %i[code label expires_at enabled]
  },
  {
    name: :effects,
    model: Effect,
    order: :id,
    columns: %i[key name enabled position]
  }
].freeze

EXPORT_TARGETS.each do |target|
  rows = target[:model].order(target[:order]).map do |record|
    target[:columns].each_with_object({}) do |column, attrs|
      attrs[column] = record.public_send(column) if record.respond_to?(column)
    end
  end

  path = output_dir.join("#{target[:name]}.yml")
  File.write(path, rows.to_yaml)
  puts "exported #{target[:name]}: #{rows.size} -> #{path}"
end
```

## seedでの復元

`db/seeds/master_data.rb`

```ruby
require "yaml"

seed_dir = Rails.root.join("db", "seed_exports")

SEED_TARGETS = [
  {
    name: :referral_codes,
    model: ReferralCode,
    file: "referral_codes.yml",
    find_by: %i[code]
  },
  {
    name: :effects,
    model: Effect,
    file: "effects.yml",
    find_by: %i[key]
  }
].freeze

SEED_TARGETS.each do |target|
  path = seed_dir.join(target[:file])
  next unless File.exist?(path)

  rows = YAML.load_file(path) || []

  rows.each do |attrs|
    finder = target[:find_by].to_h { |k| [k, attrs.fetch(k)] }
    record = target[:model].find_or_initialize_by(finder)
    record.update!(attrs)
  end

  puts "seeded #{target[:name]}: #{rows.size}"
end
```

---

## seeds.rb

```ruby
load Rails.root.join("db/seeds/master_data.rb")
```

---

## 補足

* `id` は退避しない
* `created_at / updated_at` も不要
* 業務データは対象外
* `find_by` はユニークキーを指定すること
