# 07_モード導線設計（Phase1 / #208）

## 0. 目的

[Epic #207] モード導線統一に基づき、現行 routes / 現行構成に **100%準拠**して導線仕様を確定する。

* root = ブース一覧（全ロール共通）
* ブースクリック導線の一本化（enter導線）
* cast本人は即配信（ワンクリック）
* store_admin / system_admin は視聴 or 配信を選択（※現状 modal 基盤は未実装 → 本設計で追加方針を確定）
* current_booth / current_store の整合性ルール確定
* ヘッダー / メイン / フッター のレイアウト統一（デザインは後続Issue）
* ポイント表示と購入導線の配置確定

**重要制約**

* URL直書き禁止。遷移は path helper で統一する。
* BAN / archived 除外など既存フィルタを壊さない。

## 2. レイアウト設計（確定）

※ デザインは後続Issueで行う。ここでは構造のみ「ヘッダー / メイン / フッター」を統一する。

### 2-1. ヘッダー（共通）

表示項目：

* ページタイトル（`content_for(:title)` を優先）
* ユーザー表示（表示名 or email）
* current_store 表示（存在する場合）
* current_booth 表示（存在する場合）
* ポイント残高表示（wallet.available_points）

アカウントメニュー（右側、クリックでドロップダウン想定・実装は後続IssueでもOK）：

* メールアドレス
* role
* current_store（参照のみ）
* current_booth（参照のみ）
* ログアウト

**表示は参照用とし、ここでは切替操作は行わない。**

### 2-2. フッター（共通トップレベル導線）

導線（左→右）：

* ホーム：`root_path`
* お気に入り：`favorites_booths_path`（customer前提。未ログインはログインへ）
* **中央：配信（cast最短入口）**
* ポイント購入（POST）
* ダッシュボード（role別）

#### 配信ボタン表示条件

* `current_user.present? && current_user.at_least?(:cast)` の場合のみ表示

#### 配信ボタン遷移先

* 常に `cast_booths_path`

  * 理由：routes上 cast入口は `cast_booths_path` であり、現行UIも「ブースを選択」ページが入口。
  * “即配信” は enter 導線で `live_cast_booth_path(booth)` へ飛ばすことで実現する。

#### ポイント購入（フッター）

* 現行 routes は `wallet_purchases_path` が **POSTのみ**。
* **フッターからの購入は booth_id を付けない**（文脈が無いので戻り先を root に固定）

  * `CheckoutController#return` は `@booth_id` が無い場合 `root_path` に戻る実装で整合している。
* ブース視聴ページ内の購入導線（booth_id付き）は現行維持。

  * booth内購入はブースに戻れる（booth_id付き）

#### ダッシュボード（role別）

* system_admin：`system_admin_root_path`
* store_admin：`admin_root_path`
* cast：`cast_booths_path`（当面の “castの入口/ダッシュボード相当”）
* customer：`root_path`（またはお気に入りへ誘導しても良いが最小差分は root）

## 4. enter導線の仕様（最重要）

### 4-0. 追加するルート（helper名で統一）

* `enter_booth_path(booth)`（GET）
* `enter_as_cast_booth_path(booth)`（POST）

（routes例：`resources :booths do member { get :enter; post :enter_as_cast } end`）

### 4-1. customer

* `booth_path(booth)` へリダイレクト（視聴）
* BAN防衛：

  * Home/Favoritesは「可能な範囲で予防」
  * 最終防衛は `BoothsController#show` の `StoreBanGuard` が担保
  * enter では最小差分として “showに委ねる” でもOK（ただしUX上は enter側で弾いてもよい。実装Issueで選択）

### 4-2. cast（最優先：即配信）

条件：`BoothCast.exists?(cast_user_id: current_user.id, booth_id: booth.id)` が true（= 自分のブース）

true の場合：

* session更新

  * `session[:current_booth_id] = booth.id`
  * `session[:current_store_id] = booth.store_id`
* リダイレクト先（即配信）：

  * `live_cast_booth_path(booth)`

    * 理由：現行 cast は `live` 画面が standby/live/away を扱う入口で、offlineの場合は `live` で summaryへ戻す/エラー表示になる
* モーダル表示なし（即配信のため）

false（自分のブースではない）の場合：

* `booth_path(booth)`（視聴）へリダイレクト

### 4-3. store_admin

自店ブース判定：`current_user.admin_of_store?(booth.store_id)` が true の場合

true の場合：

* 「視聴 / 配信」を選択させる（モーダル想定）

  * 視聴：`booth_path(booth)`

    * ※ `watch_admin_booth_path` は最終的に `booth_path` へリダイレクトするガード用途のため、モーダルの視聴ボタンは `booth_path` でOK（最小差分）
  * 配信：`enter_as_cast_booth_path(booth)`（POST）→ cast配信へ
    false の場合：
* `booth_path(booth)`（視聴）へリダイレクト

### 4-4. system_admin

* 常に「視聴 / 配信」を選択（モーダル想定）

  * 視聴：`booth_path(booth)`
  * 配信：`enter_as_cast_booth_path(booth)`（POST）

## 5. POST enter_as_cast（配信として入る）

### 目的

store_admin/system_admin が “配信” を選んだ場合、cast側へ安全に入るための共通導線。

### 認可

* system_admin：OK
* store_admin：自店ブースのみOK（`admin_of_store?`）
* cast：BoothCastありのみOK

### 認可OKの場合の動作

* session更新

  * `session[:current_booth_id] = booth.id`
  * `session[:current_store_id] = booth.store_id`
* `live_cast_booth_path(booth)` へリダイレクト

---

## 6. current_booth / current_store 整合性設計（確定）

### セッションキー

* `session[:current_booth_id]`
* `session[:current_store_id]`

### 整合性ルール（必須）

1. current_booth が存在する場合

* current_store は `current_booth.store` を優先する
* `session[:current_store_id]` が不一致なら、storeを補正する（必要なら sessionも補正）

2. current_booth が無効の場合（record不存在 / 権限的に無効）

* `session[:current_booth_id]` をクリアする

3. admin側で store を切り替えた場合（`admin_current_store_path`）

* 事故防止のため `session[:current_booth_id]` をクリアする

  * 実装箇所：`Admin::CurrentStoresController#create`（最小差分で完結）

## 7. モーダル設計（現状なし → 追加方針を確定）

現状：modal関連は未実装。
#208では「store_admin/system_admin は視聴 or 配信を選択」が必須なので、モーダル基盤を追加する。

方針：

* layoutに共通modal枠を1つ配置
* `<turbo-frame id="modal">` を常設（空でOK）
* `enter` が store_admin/system_admin 向けに “モーダル本文” を返す（HTML partial）
* 遷移ボタンは `_top` で画面遷移（安全優先）

  * 視聴：`booth_path(booth)`（GET）
  * 配信：`button_to enter_as_cast_booth_path(booth)`（POST）を `_top` で実行

※ 実装は後続Issueで行う（本Issueは設計確定）

## 8. テスト観点（必須）

* enter分岐の網羅

  * customer → booth_path
  * cast本人（BoothCastあり）→ session更新 + live_cast_booth_path
  * cast他人 → booth_path
  * store_admin 自店 → モーダル本文（または設計どおりのHTML）
  * store_admin 他店 → booth_path
  * system_admin → モーダル本文
* enter_as_cast

  * system_admin OK
  * store_admin 自店OK / 他店NG
  * cast BoothCastありOK / なしNG
  * session更新 + live_cast_booth_path
* Current整合

  * boothがある時に store不一致なら補正される
  * admin store切替で boothセッションがクリアされる
* フッター配信ボタン

  * `at_least?(:cast)` で表示/非表示
* フッター購入導線

  * booth_id無し→ returnはrootへ
  * booth内購入（booth_idあり）→ returnはboothへ
